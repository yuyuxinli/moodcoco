# Viral Adaptation

将已验证的爆款文章拆解并改造为心情可可公众号的本土化选题方案。

## 定位与边界

**做什么**：从一篇已爆的参考文章出发，提炼可复用机制，生成本土化改造方案
**不做什么**：不写正文、不评分选题、不做结构大纲（这些交给下游 skill）

**输入**：一篇已爆的参考文章（URL、文件路径、或直接粘贴内容）
**输出**：改造方案（可直接喂给 topic-evaluation + article-blueprint）

**与其他 skill 的关系**：

```
本 skill（改造方案）
    → topic-evaluation（评分验证）
    → article-blueprint（结构规划）
    → 写正文
```

- 本 skill 是上游，输出格式已对齐下游接口
- `topic-evaluation` → 用本 skill 输出的「选题评分卡输入」直接评分
- `article-blueprint` → 用本 skill 输出的「蓝图输入」直接进入 Step 0

## Context

读取 `.claude/product-marketing-context.md` 获取产品定位、目标人群、品牌语言。

读取 `references/adaptation-dimensions.md` 获取改造维度参考清单。

## 核心原则

**机制优先**：不是"翻译"原文，是提取让它爆的底层机制（心理钩子、结构节奏、信息差），然后用全新的内容重建。

**逐步交互**：每步完成后暂停，等待用户确认/调整后再进入下一步。绝不跳步。

**保留骨架，替换血肉**：结构节奏、心理机制、情绪曲线形状可以借鉴；案例、数据、文化语境、表达方式必须替换。

**原创性红线**：最终产出不能让读者联想到原文。如果原文是英文，中文读者本来就不知道原文；如果原文是中文，改造幅度必须更大。

## Workflow

```
Step 1: 解剖 — 拆解参考文章的骨架
       ↓ [用户确认]
Step 2: 诊断 — 为什么它能爆？归因到可复用机制
       ↓ [用户确认]
Step 3: 差异映射 — 原文受众 vs 我们的受众，逐项对比
       ↓ [用户确认]
Step 4: 改造方案 — 输出本土化后的选题+角度+替换清单
       ↓ [用户确认]
Step 5: 交接 — 生成 topic-evaluation 输入 + article-blueprint 输入
       ↓ [输出完整改造方案]
```

### Step 1: 解剖

**目的**：把参考文章拆成可分析的零件。

**获取原文**：
- 如果是 URL：用浏览器工具获取全文
- 如果是文件路径：直接读取
- 如果是粘贴内容：直接使用

**拆解为以下维度**：

#### 1a. 结构骨架

列出文章的章节/段落结构，每段用一句话概括：

```
| # | 段落/章节 | 一句话概括 | 叙事功能 |
|---|----------|-----------|---------|
| 1 | [标题/位置] | [在说什么] | [钩子/铺垫/冲击/证据/转折/收束] |
| 2 | ... | ... | ... |
```

**叙事功能标签**：
- **钩子**：开头抓注意力
- **铺垫**：建立背景和代入感
- **冲击**：信息差或认知颠覆
- **证据**：数据、案例、亲身经历
- **拦截**：预判读者的"但是..."并回应
- **转折**：视角切换或情绪转向
- **升华**：从具体拉到更高维度
- **收束**：结尾，给出口或行动号召

#### 1b. 心理钩子清单

列出文章中使用的所有心理机制：

```
| # | 位置 | 心理机制 | 具体表现 |
|---|------|---------|---------|
| 1 | 开头 | [如：类比锚定] | [如：用 COVID 类比 AI 变革] |
| 2 | ... | ... | ... |
```

常见心理机制参考：
- 类比锚定（用已知的重大事件类比未知的）
- 信息差冲击（"你不知道的事实"）
- 身份代入（"我也是普通人，我先被冲击了"）
- 权威信号（行业身份、数据引用）
- 恐惧唤醒（FOMO、被替代焦虑）
- 认知失调（两个都对的东西互相矛盾）
- 具体化锚点（一个极具体的场景/数字）
- 社会证明（"很多人已经..."）
- 紧迫感（"不是将来，是现在"）

#### 1c. 数据锚点

列出文章中的关键数据/事实：

```
| # | 数据/事实 | 来源 | 功能 | 可替换性 |
|---|----------|------|------|---------|
| 1 | [如：METR 测试结果] | [如：METR 官方报告] | [如：证明 AI 能力跃升] | 可替换/需保留/需补充 |
```

**输出格式**：

```
## Step 1: 解剖结果

### 结构骨架
（表格）

### 心理钩子清单
（表格）

### 数据锚点
（表格）

### 原文基本信息
- 标题：
- 作者：
- 平台：
- 语言：
- 大约字数：
- 发布时间（如可知）：
- 传播量级（如可知）：
```

### Step 2: 诊断

**目的**：回答"为什么这篇文章能爆"，归因到可复用的机制，而不是不可复制的因素。

**分析框架**：

#### 2a. 爆点归因

从以下维度逐一分析，标注"可复用"或"不可复用"：

```
| 维度 | 原文表现 | 可复用? | 说明 |
|------|---------|--------|------|
| 开头钩子 | [如：COVID 类比] | 可复用 | [类比结构可用，具体类比物需替换] |
| 叙事结构 | [如：先个人→再行业→再普通人] | 可复用 | [由近及远的结构有效] |
| 身份设定 | [如：行业内部人士] | 部分可用 | [需要找到我们的"内部人"角色] |
| 核心情绪 | [如：紧迫感+赋能] | 可复用 | [情绪配比可以调整] |
| 传播动机 | [如：信息差分享] | 可复用 | [读者转发的心理动机] |
| 时机因素 | [如：新模型刚发布] | 不可复用 | [时间窗口已过] |
| 作者光环 | [如：6年AI创业者] | 不可复用 | [需要替换为我们的信源] |
```

#### 2b. 情绪曲线还原

用简图还原原文的情绪走向：

```
高 ●────────────────────────●
   │
   │     X
   │  X     X        X
   │X          X   X
   │              X
低 ●────────────────────────●
   开头  中段        结尾
```

标注每个转折点对应的段落和心理机制。

#### 2c. 一句话诊断

用一句话总结"它为什么能爆"：

格式：`它能爆，因为 [核心机制]，具体表现为 [可感知的读者反应]`

**输出格式**：

```
## Step 2: 诊断结果

### 爆点归因
（表格）

### 情绪曲线
（ASCII 图 + 标注）

### 一句话诊断
[一句话]

### 可复用机制清单（按优先级排序）
1. [最重要的可复用机制]
2. [次要的可复用机制]
3. ...
```

### Step 3: 差异映射

**目的**：逐项对比原文受众和我们的受众，列出保留/替换/删除清单。

**读取 `references/adaptation-dimensions.md`** 获取改造维度参考。

**逐维度对比**：

```
| 改造维度 | 原文 | 我们 | 操作 | 具体方案 |
|---------|------|------|------|---------|
| 受众 | [如：美国科技从业者] | [如：18-24岁高敏感女大学生] | 替换 | [如何替换] |
| 作者身份 | [如：AI创业者] | [如：心情可可/可可] | 替换 | [新的身份设定] |
| 核心恐惧/痛点 | [如：被AI替代] | [如：情绪内耗] | 替换 | [对应的痛点] |
| 案例/故事 | [如：作者自己写代码] | [如：？] | 替换 | [需要什么样的新故事] |
| 数据锚点 | [如：METR数据] | [如：？] | 保留/替换/补充 | [具体处理] |
| 情绪框架 | [如：恐惧→紧迫→赋能] | [如：？] | 调整 | [适合我们受众的情绪配比] |
| 文化语境 | [如：英文长文] | 公众号分章节 | 适配 | [格式/敏感词/平台规则] |
| 身份安全 | [如：转发=关心未来] | [如：转发=？] | 设计 | [读者转发时的社交形象] |
| 结尾走向 | [如：警告+行动号召] | [如：？] | 调整 | [适合我们的结尾策略] |
| 产品衔接 | [原文无] | 心情可可 | 新增 | [在哪里、如何自然衔接] |
```

**关键提问**（在此步骤向用户确认）：

1. 你有没有可以替换的真实案例/故事？
2. 改造后的目标是哪个 persona？（从 product-marketing-context 的 3 个 persona 中选）
3. 情绪框架要偏哪个方向？（恐惧驱动 / 好奇驱动 / 赋能驱动 / 共鸣驱动）

**输出格式**：

```
## Step 3: 差异映射

### 逐维度对比
（表格）

### 需要用户提供的素材
- [ ] [素材1：如"有没有真实用户案例"]
- [ ] [素材2：如"你自己在这个话题上的经历"]

### 改造难度评估
- 整体难度：低/中/高
- 最大挑战：[一句话说明]
- 建议：[一句话建议]
```

### Step 4: 改造方案

**目的**：输出完整的本土化改造方案，包括新选题、新角度、替换清单。

#### 4a. 新选题定义

```
### 新选题
- 主题：[一句话描述]
- 角度：[独特切入点]
- 目标读者：[具体的一个人]
- 情绪旅程：从 [起点] → 到 [终点]
```

#### 4b. 结构改造对照

逐章对比原文结构和改造后的结构：

```
| # | 原文段落功能 | 改造后的段落功能 | 保留的机制 | 替换的内容 |
|---|------------|---------------|----------|----------|
| 1 | [如：COVID类比开头] | [如：用X类比开头] | 类比锚定 | 类比物替换 |
| 2 | ... | ... | ... | ... |
```

#### 4c. 案例替换清单

```
| # | 原文案例 | 替换为 | 替换理由 | 素材来源 |
|---|---------|-------|---------|---------|
| 1 | [如：作者周一写代码] | [如：某同学用AI做课设] | [受众代入感] | [需用户提供/可虚构/来自竞品数据] |
```

#### 4d. 情绪框架调整

```
原文情绪配比：恐惧 60% + 紧迫 20% + 赋能 20%
改造后配比：[如：共鸣 40% + 好奇 30% + 赋能 30%]

调整理由：[一句话]
```

#### 4e. 身份安全检查

回答：读者转发这篇改造后的文章时，她的朋友圈会怎么看她？

```
转发形象：[如："关注前沿的信息灵通人士" / "有情绪觉察的成熟女生"]
转发动机：[如：信息差分享 / 情绪共鸣 / 身份认同]
潜在风险：[如：是否会让转发者显得焦虑/负面？如何规避]
```

**输出格式**：

```
## Step 4: 改造方案

### 新选题定义
（如上）

### 结构改造对照
（表格）

### 案例替换清单
（表格）

### 情绪框架调整
（配比 + 理由）

### 身份安全检查
（转发形象 + 动机 + 风险）
```

### Step 5: 交接

**目的**：将改造方案格式化为下游 skill 可直接使用的输入。

#### 5a. topic-evaluation 输入

生成一份可以直接喂给 `topic-evaluation` skill 的选题描述：

```
### 给 topic-evaluation 的输入

**选题**：[改造后的选题，一句话]
**角度**：[具体的切入角度]
**目标读者**：[persona]
**心理钩子**：[主要的心理机制]
**传播动机**：[读者为什么会转发]
**参考来源**：[改造自哪篇文章，简要说明]
```

#### 5b. article-blueprint 输入

生成一份可以直接喂给 `article-blueprint` Step 0 的聚焦结果：

```
### 给 article-blueprint 的输入

【聚焦结果】
- 读者：[具体的一个人]
- 场景：[她正在经历的具体瞬间]
- 情绪旅程：从 [起点情绪] → 到 [终点情绪]
- 切入角度：[一句话说明独特切入点]

【改造参考】
- 可复用结构：[简要描述借鉴的结构节奏]
- 可复用机制：[简要描述借鉴的心理机制]
- 案例素材：[已确定的替换案例列表]
- 情绪配比：[目标情绪配比]
```

#### 5c. 原创性自检

在交接前进行最终检查：

```
### 原创性自检
- [ ] 标题与原文无相似度
- [ ] 开头手法已替换（不是翻译原文开头）
- [ ] 所有案例/故事已替换为本土内容
- [ ] 数据锚点已替换或标注来源
- [ ] 结尾走向已调整为适合我们受众的方向
- [ ] 中文读者不会联想到原文
```
