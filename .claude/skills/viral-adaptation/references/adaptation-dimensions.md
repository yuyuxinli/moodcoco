# 改造维度参考清单

基于真实案例提炼的"爆款文章本土化改造"维度参考。

## 真实案例基础

**原文**：Matt Shumer "Think back to February 2020"（英文长文，科技圈爆款）
**改造后**："同一个人用不同的 AI，产出差了 140 倍"（心情可可公众号，微信生态爆款）

---

## 改造维度

### 1. 受众替换

**核心问题**：原文写给谁？我们写给谁？

| 原文受众特征 | 改造方向 | 注意事项 |
|------------|---------|---------|
| 职业身份 | 替换为目标 persona 的身份 | 不能太泛（"年轻人"），要具体到一个人 |
| 知识背景 | 调整前置知识假设 | 原文假设读者知道的，我们的读者可能不知道 |
| 核心焦虑 | 替换为目标读者的焦虑 | 焦虑的具体形态不同，本质机制可能相同 |
| 日常场景 | 替换为目标读者的生活场景 | 办公室 → 宿舍、实验室 → 图书馆 |

**Matt → 公众号案例**：
- 美国科技从业者 → 中国大学生/职场新人
- "被 AI 替代" → "信息差焦虑 + 如何用好 AI"
- 办公室写代码 → 课堂/实习/求职

### 2. 作者身份替换

**核心问题**：原文作者凭什么被信任？我们凭什么被信任？

| 原文信源 | 改造策略 | 示例 |
|---------|---------|------|
| 行业大佬亲述 | 找对应领域的"内部人" | 6年AI创业者 → 实际使用AI完成工作的团队 |
| 个人经历 | 替换为我们/用户的真实经历 | "我周一写代码" → "我们团队用AI做营销" |
| 权威数据 | 找对应的本土数据 | METR报告 → 国内AI使用调研 |

**关键原则**：信任建立的机制可以借鉴（"我不是预测，是亲历"），但具体的信任来源必须替换。

### 3. 案例/故事替换

**核心问题**：原文用什么故事打动人？我们用什么替代？

| 案例类型 | 替换策略 | 素材来源 |
|---------|---------|---------|
| 作者亲身经历 | 替换为我们的真实经历 | 团队日常、用户反馈 |
| 行业数据 | 替换为本土数据 | 国内报告、自己跑的数据 |
| 名人案例 | 替换为目标读者可代入的案例 | 竞品用户案例、公开报道 |
| 虚构场景 | 替换为目标读者的生活场景 | persona 分析中的场景标签 |

**Matt → 公众号案例**：
- "我周一让AI写代码" → "同一个人用不同AI，产出差140倍"（自己跑的真实数据）
- "GPT-5.3 + Opus 4.6 同日发布" → 保留（事实性内容可保留）
- "律师、医生、会计" → "课设、论文、实习汇报"（场景替换）

### 4. 情绪框架调整

**核心问题**：原文用什么情绪驱动传播？我们需要调整吗？

| 情绪类型 | 适用场景 | 风险 |
|---------|---------|------|
| 恐惧驱动 | 重大变革期、信息差大 | 可能引发焦虑反感 |
| 好奇驱动 | 新鲜信息、反直觉发现 | 可能浅层停留 |
| 共鸣驱动 | 普遍经历、未被命名的感受 | 可能缺乏新鲜感 |
| 赋能驱动 | 给出行动方案、工具 | 可能变成说教 |
| 机会驱动 | 信息差 = 先机 | 需要真的有干货 |

**情绪配比调整原则**：
- 公众号传播靠"转发"和"在看"，情绪共鸣 > 纯信息差
- 恐惧驱动的文章打开率高但转发率不一定高（转发 = 社交形象风险）
- 最佳组合通常是：好奇（打开）+ 共鸣/赋能（转发）

**Matt → 公众号案例**：
- 原文：恐惧 60% + 紧迫 20% + 赋能 20%
- 改造后：机会 40% + 好奇 30% + 赋能 30%（从"你要被替代"变为"信息差就是机会"）

### 5. 数据锚点处理

**核心问题**：原文的数据要保留、替换还是补充？

| 处理策略 | 适用条件 | 示例 |
|---------|---------|------|
| 保留 | 数据本身有冲击力且来源可靠 | METR 测试结果（国际公认） |
| 替换 | 数据对目标读者无感 | 美国就业数据 → 国内就业数据 |
| 补充 | 原文缺少但我们有的独家数据 | 自己跑的"140倍"数据 |
| 删除 | 数据过时或不可验证 | 过期的市场预测 |

**关键**：自己跑出来的独家数据 > 引用别人的数据。独家数据是改造后文章的核心差异化来源。

### 6. 文化适配

**核心问题**：原文的文化语境在目标市场能否通？

| 维度 | 检查点 | 示例 |
|------|-------|------|
| 平台格式 | 文章长度、分段、标题层级 | 英文长文 → 公众号分章节、加粗重点 |
| 敏感词 | 微信生态敏感词检查 | 参考 CLAUDE.md 中的敏感词替代表 |
| 文化梗 | 原文的文化引用是否需要替换 | "cocktail party" → 不需要直译 |
| 阅读习惯 | 开头节奏、信息密度 | 英文可以铺垫长，公众号前3屏定生死 |
| 法律合规 | 原文提及的产品/公司是否敏感 | 某些AI产品在国内可能需要模糊化处理 |

### 7. 身份安全

**核心问题**：读者转发这篇文章时，她的朋友圈会怎么看她？

| 转发形象 | 吸引力 | 风险 |
|---------|-------|------|
| 信息灵通人士 | 高 | 可能显得焦虑 |
| 有深度思考的人 | 高 | 可能显得装 |
| 有情绪觉察的人 | 中高 | 较安全 |
| 关心朋友的人 | 高 | 很安全 |
| 求助/倾诉 | 低 | 暴露脆弱 |

**安全公式**：`转发文案暗示的形象` 必须是读者**愿意展示**的形象。

**Matt → 公众号案例**：
- 原文转发形象："我关心你的未来"（安全）
- 改造后转发形象："我比你早知道这个信息"（安全+优越感）

### 8. 结尾走向

**核心问题**：原文怎么收尾？我们应该怎么收？

| 结尾类型 | 适用场景 | 效果 |
|---------|---------|------|
| 警告 | 重大风险话题 | 唤起行动但可能引发焦虑 |
| 行动号召 | 有明确下一步的 | 转化率高但可能感觉硬 |
| 赋能 | 读者需要信心的 | 温暖但可能缺乏紧迫感 |
| 开放出口 | 话题没有标准答案 | 余韵好但可能感觉没说完 |
| 情绪出口 | 情绪类文章 | 被理解感强，转发意愿高 |

**公众号结尾最佳实践**：
- 正文结尾用"赋能"或"开放出口"
- 产品衔接放 PS 区域，永远是"如果你需要"而非"快来用"
- 最后一句话要值得截图发朋友圈

## 改造质量检查清单

改造完成后，用以下问题自检：

- [ ] **不是翻译**：如果把改造后的文章翻译回原文语言，读起来像一篇新文章吗？
- [ ] **受众对了**：目标读者读前 3 屏会觉得"这说的是我"吗？
- [ ] **故事是新的**：所有案例/故事都是目标读者能代入的本土内容吗？
- [ ] **数据有独家性**：有至少一个别的文章没有的数据/洞察吗？
- [ ] **情绪适配了**：情绪框架适合目标读者和目标平台吗？
- [ ] **转发安全**：读者转发时的社交形象是正面的吗？
- [ ] **结尾合适**：结尾走向适合我们的品牌调性吗？
- [ ] **产品能衔接**：有自然的产品衔接点吗？
- [ ] **敏感词干净**：微信生态敏感词都避开了吗？
