# Article Blueprint

为心情可可公众号叙事型纯文字长文生成**写作蓝图**，基于雪花写作法（Snowflake Method）改编。

## 定位与边界

**做什么**：从模糊方向递进式展开为完整的写作结构规划
**不做什么**：不写正文、不做多卡图文、不做选题评分、不生成出图指令

**输入**：一个模糊方向（如"想写点关于毕业季的"）或一个明确选题
**输出**：一份可以直接交给人写正文的「写作蓝图」

**与其他 skill 的关系**：
- `topic-evaluation` → 选题评分通过后，可进入本 skill
- 本 skill → 输出写作蓝图
- 人/AI → 基于蓝图写正文
- `visual-story-prompts` → 如需配图另走
- `eg-emotion-knowledge` → 可选，当文章涉及情绪命名/辨析时调用

## Context

读取 `.claude/product-marketing-context.md` 获取产品定位、目标人群、品牌语言。

## 核心原则

**递进式展开**：每一步都是前一步的自然扩展，像放大镜一样逐层聚焦。

**逐步交互**：每步展开后暂停，等待用户确认/调整后再进入下一步。绝不跳步。

**叙事型专用**：所有结构设计都围绕"用场景和故事带读者进入情绪"这一核心手法，不适用于干货列点文。

**公众号适配**：雪花原版为长篇小说设计，本 skill 为 2000-5000 字公众号文章适配。"灾难"不是外部事件爆炸，是内心的认知冲击——读者发现自己一直以来的某个想法是错的。

## Workflow

```
Step 0: 聚焦 — 从模糊方向 → 明确"写给谁 + 写什么 + 切入角度"
        ↓ [用户确认]
Step 1: 一句话 — 用 ≤25 字概括文章的核心张力
        ↓ [用户确认]
Step 2: 骨架句 — 每章一句话，展开全文结构
        ↓ [用户确认]
Step 3: 人物卡 — 读者代理人 + 叙述者
        ↓ [用户确认]
Step 4: 章节大纲 — 展开为 5-7 个章节
        ↓ [用户确认]
Step 5: 场景清单 — 情绪曲线 + 产品衔接点 + 不可复用检查
        ↓ [输出完整蓝图]
```

---

### Step 0: 聚焦

**目的**：将模糊方向转化为可以动笔的明确选题。

**如果输入已经很明确**（有具体角度和目标读者），可以跳过本步直接进入 Step 1。

**如果输入是模糊方向**，用 3 个提问逐步聚焦：

**提问 1：写给谁？**

从 product-marketing-context 的 3 个 persona 中选择或描述：

| Persona | 核心关切 |
|---------|---------|
| 高敏感女大学生 | 被理解、被看见、情绪安全感 |
| 恋爱中的女生 | 亲密关系中的安全感 |
| 考研/求职期女生 | 确定性、自我价值感 |

不限于这 3 个，但必须具体到"一个人"，不能是"年轻人"或"女性"。

**提问 2：她在什么具体场景下会有这个感受？**

目标：从抽象主题锚定到一个可感知的瞬间。

- 不好：「毕业季的迷茫」
- 好：「答辩结束后在食堂吃最后一碗面，突然不知道明天该去哪」

**提问 3：读完这篇文章，她应该获得什么？**

不是知识层面的收获，而是情绪层面的变化：

- 不好：「了解如何应对迷茫」
- 好：「从'我是不是不够好'变成'还没找到，但我敢去找了'」

**输出格式**：

```
【聚焦结果】
- 读者：[具体的一个人]
- 场景：[她正在经历的具体瞬间]
- 情绪旅程：从 [起点情绪] → 到 [终点情绪]
- 切入角度：[用一句话说明这篇文章独特的切入点]
```

---

### Step 1: 一句话

**目的**：用 ≤25 个字概括文章的核心张力。

**必须包含**：
- 一个具体的人（不是"年轻人"，是"刚毕业的你"或"深夜失眠的你"）
- 一个具体的困境或矛盾（内在的拉扯，不是外部障碍）
- 不泄露结局

**张力来源参考**（选一个或组合）：

| 张力类型 | 模式 | 例子 |
|---------|------|------|
| 反直觉 | 你以为 A，其实是 B | "票好买了你反而不想回了" |
| 认知失调 | 两个真实的东西互相矛盾 | "AI 写了 100% 的代码，但没造出产品" |
| 命名未命名 | 给一种模糊感受第一次命名 | "你不是懒，是害怕开始后又失败" |
| 隐藏代价 | 看似正确的选择有隐藏的代价 | "你学会了懂事，但你失去了生气的能力" |

**输出格式**：

```
【一句话】
[≤25 字的核心张力]

【张力类型】：[类型名]
【为什么这个张力能抓住目标读者】：[一句话解释]
```

---

### Step 2: 骨架句

**目的**：将一句话展开为全文骨架——每章一句话，合起来就是文章的完整叙事弧。

**句数 = 章节数**。雪花写作法原版固定五句话（背景→灾难1→转折→灾难2→结局），但公众号文章章节数不固定。本步骤根据文章实际需要，先确定章节数，再为每章写一句骨架句。

**每句骨架句的要求**：
- 一句话概括该章的核心内容（不是标题，是"这章在说什么"）
- 标注该章的**叙事功能**（从下方功能库中选）
- 标注该章的**情绪目标**（读者读完这章应该有什么感觉）

**叙事功能库**（根据需要选用，可组合）：

| 功能 | 公众号叙事文中的表现 | 典型情绪目标 |
|------|-------------------|------------|
| 立场景 | 读者的日常场景，制造代入感 | "这说的是我" |
| 认知冲击 | 第一个让读者"愣住"的发现 | "等等，好像哪里不对" |
| 拦截怀疑 | 预判并回应读者的"但是..." | "好吧，我的反驳站不住" |
| 立框架 | 引入新的分析视角 | "原来可以这样看" |
| 深层击中 | 更深层的真相，读者被戳到 | "说的就是我" |
| 信息差冲击 | 展示读者不知道的事实/世界 | "居然还有这些我不知道的" |
| 情绪出口 | 不给答案，打开一个出口 | "虽然还没想通，但好像可以了" |
| 升华/收束 | 把具体观点拉到更高维度 | "原来这件事的本质是..." |

功能库不是穷举，遇到新的叙事功能可以自行命名。

**关键适配说明**：

- **认知冲击 ≠ 坏事**。在公众号叙事文中，"冲击"是读者发现自己一直以来的某个想法/应对方式是有代价的
- **结局不是解决问题**。公众号文章的结局通常不给答案，而是打开一个出口——"你不需要今天想通，但说出来会好一点"
- **节奏要有起伏**。不能每章都"冲击"，需要有铺垫和呼吸的章节

**输出格式**：

```
【骨架句】（共 N 章）

1. [章节名]（功能：XX）：... 【情绪：...】
2. [章节名]（功能：XX）：... 【情绪：...】
...

【道德前提】：[读者从什么错误信念转向什么新认知]
```

---

### Step 3: 人物卡

**目的**：为文章中的主要角色建立清晰的内在逻辑。

公众号叙事文有两类角色：

#### 读者代理人（"你"）

大多数段落的视角人物。读者通过她进入故事。

```
【读者代理人】
- 谁：[具体身份，如"大四在找工作的女生"]
- 日常世界：[她的日常状态是什么样的]
- 核心困境：[她现在卡在什么地方]
- 错误信念：[她以为是怎样的（但其实不对）]
- 顿悟：[她在文章结尾处意识到了什么]
- 场景标签：[3-5 个她会出现的具体场景，如"宿舍上铺、图书馆、招聘会"]
- 口头禅/内心 OS：[她会说/想的具体句子，用来保持语言一致性]
```

#### 叙述者（"我"或可可）

提供转折和洞察的声音。

```
【叙述者】
- 身份：[可可 / 创始人 / 第三方视角]
- 资格：[为什么她有资格谈这个——亲身经历？陪伴过 300 个用户？]
- 独特视角：[她看到了读者看不到的什么]
- 出场时机：[在文章的哪个位置切入叙述，通常是转折处]
```

**注意**：如果文章中还有其他角色（如读者的妈妈、男朋友、室友），不需要完整人物卡，但要在场景清单中标注他们的功能（是催化剂、镜子、还是对照组）。

---

### Step 4: 章节大纲

**目的**：将骨架句展开为详细的章节大纲。

**章节格式**（与已发布文章一致：一、二、三...）：

```
### 章节 [N] — [章节名]

**视角**：[读者"你" / 叙述者"我"]
**内容概述**：[一段话，50-100 字]
**情绪目标**：[读者读完这章应该有什么感觉]
**与上一章的关系**：[承接/对比/升级/转折]
```

**章节节奏规则**：

1. **开头 1-2 章**必须是读者"你"的视角，用场景拉进代入感，不要急于切换到叙述者
2. **转折处**（通常第 3 章前后）切换到叙述者视角
3. **高潮处**（通常倒数第 2 章）是情绪最强的段落
4. **结尾章**通常回到"你"，或者叙述者对"你"说话

**避免**：
- 章节之间情绪平直（每章必须比前一章有情绪变化）
- 连续 3 个章节同一视角（要有切换）
- 开头章节就出现说教/观点输出

---

### Step 5: 场景清单 + 情绪曲线

**目的**：将章节拆解为具体场景，标注全文情绪走向和产品衔接点。

#### 5a. 场景清单

每章拆解为 1-3 个场景。每个场景标注：

```
| # | 章节 | 场景 | 类型 | 环境 | 关键细节 | 情绪 |
|---|------|------|------|------|---------|------|
```

**场景类型**（改编自雪花写作法）：

| 类型 | 构成 | 公众号中的表现 |
|------|------|--------------|
| 主动场景 | 目标→冲突→挫折 | 读者在做一件事，遇到了阻碍 |
| 被动场景 | 反应→困境→决定 | 读者消化刚发生的事，做出选择 |

**每个场景必须有冲突或张力**。如果一个场景只是"介绍背景"或"解释概念"而没有冲突，要么加上冲突，要么删掉它。

#### 5b. 情绪曲线

用 ASCII 图展示全文情绪走向：

```
高 ●─────────────────────────────●
   │
   │          X
   │      X       X
   │  X               X    X
   │X                     X   X
低 ●─────────────────────────────●
   1    2    3    4    5    6    7
```

标注每个转折点对应的章节和场景。

**情绪曲线规则**：
- 不能一路上升（读者会疲劳）
- 不能一路平直（读者会无聊）
- 高潮通常在倒数第 2 章，最后一章微降（给出口）
- 至少有一次"先降后升"的节奏（让读者在低谷被击中后再拉起来）

#### 5c. 产品衔接点

标注 2-3 个可以自然引出心情可可的位置：

```
| 位置 | 衔接方式 | 自然度 |
|------|---------|-------|
| 文末 PS | "如果深夜想聊..." | ★★★★★ |
| 章节 X 的场景 Y | 角色提及可可 | ★★★ |
| ... | ... | ... |
```

**衔接原则**：
- 永远不强推，永远是"如果你需要"
- 首选文末 PS（最自然）
- 文中提及不超过 1 次，必须是剧情需要而非广告植入
- 使用产品语言："来找可可聊聊"、"你说什么她都接着"

#### 5d. 不可复用检查

扫描 `公众号/` 目录下已发布文章，检查：
1. **结构相似度**：本文的三幕结构是否和已发布文章雷同
2. **开头场景**：是否撞了已有文章的开头手法
3. **金句/收尾**：是否有用过的句式

如果检测到重叠，列出具体内容并建议替代方案。
