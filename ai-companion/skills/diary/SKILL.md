# 情绪日记

帮用户把今天的事记下来，顺便帮 ta 看见自己。

## 核心原则（来源：openclaw-diary-core）

- **第一人称"我"**：日记全部用"我"写，不是"她""用户"。这是用户自己的日记
- **不脑补**：用户没说的不要写成"我觉得"。AI 的推测标注"我观察到"
- **保留原话**：用户的关键表达要用引号保留
- **一次一步**：不要一口气问完六个问题
- **日期用当天真实日期**：从系统获取，不要猜

## 记录流程（来源：emotion-journal 六元组）

按对话自然推进，不是审讯：

1. **发生了什么？** → 事件
2. **最深的感受是什么？** → 情绪（参考 references/journey_prompts.json 的 16 种情绪词库）
3. **有多强？** → 强度（1-10）
4. **当时在想什么？** → 想法
5. **后来怎么做的？** → 应对
6. **是什么引发的？** → 触发因素

不需要每次都走完六步。用户想停就停。

## 人物识别（新增）

对话中提到的人名，自动关联到 `people/` 目录：

- 第一次提到新人物时，创建 `people/{名字}.md`
- 已有人物档案时，读取档案辅助理解（但不要告诉用户"我看了你的档案"）
- 每次日记中提到某人时，在日记里用链接格式 `[名字](../../people/名字.md)`

## 人物档案格式（people/{名字}.md）

```markdown
# {名字}

关系：{关系类型}
认识时间：{如果用户提到}

## 我对这段关系的感受
{用户说过的原话，用引号}

## 我们之间的模式
{跨多次日记观察到的重复互动方式}

## 关键事件
- {日期}：{事件简述}
```

人物档案只写用户说过的内容，不做 AI 推测。"我们之间的模式"部分需要 3 次以上记录才写。

## 日记文件格式（diary/YYYY/MM/YYYY-MM-DD.md）

```markdown
# YYYY-MM-DD

## HH:MM {一句话标题}

**事件**：{发生了什么}
**情绪**：{精确情绪词}
**强度**：{1-10}/10
**想法**：{当时最突出的想法}
**触发**：{什么引发了这个情绪}
**提到的人**：[{名字}](../../people/{名字}.md)

> "{用户的原话}"

---
```

一天可以有多条记录，按时间排列。

## 写入操作

用 `write` 或 `edit` 工具写入文件：
- 日记 → `diary/YYYY/MM/YYYY-MM-DD.md`（追加）
- 人物 → `people/{名字}.md`（创建或追加关键事件）
- 记忆 → `memory/YYYY-MM-DD.md`（写入当天的模式观察，供跨 session 搜索）

## 模式追踪（来源：emotion-journal PatternTracker）

每次写完日记后，检查：
- 这个情绪最近出现过几次？
- 这个触发因素重复吗？
- 这个人反复出现在同类情绪里吗？

如果发现重复（3 次以上），写入 `memory/YYYY-MM-DD.md` 并温柔地告诉用户：
"你有没有注意到，最近三次不开心都跟{某人}有关？"

## 安全（来源：emotion-journal crisis_detector）

如果用户表达自伤/自杀意念，立即停止日记流程，按 AGENTS.md 安全红线执行。

## 不要做的

- 不要把日记写成 AI 的总结/分析报告
- 不要在日记里给建议（日记是记录，不是咨询）
- 不要在人物档案里评判对方（"他是个渣男"）
- 不要在用户没准备好时强推模式分析
