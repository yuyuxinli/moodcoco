# MBTI 人格测试引导

用对话式问答帮用户完成 MBTI 测试。不是填问卷，是聊天中自然地了解用户的人格倾向。

## 触发条件

- 用户进入 MBTI_GAME 类型的 session
- 用户主动说"想做个测试"、"测测我的 MBTI"、"性格测试"
- 由 AGENTS.md 路由树检测到 MBTI 意图

## 前置：加载用户上下文

1. 调用 `user_profile_get()` → 了解用户基本信息，避免重复问已知内容
2. 调用 `conversation_history({ session_id, limit: 20 })` → 获取本次测试对话历史

## 流程概览

### 阶段 1：开场引导

首次进入时，自然地介绍测试：

- 不说"欢迎来到 MBTI 测试"，而是"我们来聊聊你平时的习惯和偏好？"
- 告知大约需要回答 15 道题，每道题用 5 点量表（1=非常不同意 到 5=非常同意）
- 营造轻松氛围："没有对错，选最贴近你真实感受的就好"

### 阶段 2：题目问答

题目由前端展示（ai_quiz_practice UI Tool），Agent 负责：

1. **接收题目生成请求**：当前端需要新一批题目时
2. **引导作答**：用户回答后给予简短回应，保持对话感
3. **保存答案**：每道题作答后调用 `mbti_save_answer` 持久化

#### 答案保存

用户每回答一道题，调用：
```
mbti_save_answer({
  session_id: "<当前 session>",
  question_id: <题目 ID>,
  answer: <1-5 Likert 量表值>
})
```

返回值包含：
- `answer_message_id`：保存成功的消息 ID
- `answered_count`：已回答总数
- `is_completed`：是否达到 15 题

#### Likert 量表映射

| 值 | 含义 |
|----|------|
| 1 | 非常不同意 |
| 2 | 不同意 |
| 3 | 中性 |
| 4 | 同意 |
| 5 | 非常同意 |

得分计算由后端 Service 自动处理：
- 正向题：直接使用原始值
- 反向题：`6 - 原始值`（1→5, 2→4, 3→3, 4→2, 5→1）

### 阶段 3：完成检测

当 `mbti_save_answer` 返回 `is_completed: true` 时：

1. 告知用户"测试完成了"
2. 自然过渡到报告生成："我来帮你整理一下结果，可能需要 1-2 分钟"
3. 调用 `mbti_generate_report` 触发异步报告生成

### 阶段 4：报告生成

调用：
```
mbti_generate_report({
  session_id: "<当前 session>"
})
```

返回值包含：
- `task_id`：异步任务 ID
- `status`：任务状态（"started" 表示已启动）

报告生成是异步的，Agent 可以：
- 在等待时与用户继续聊天
- 收到报告完成事件后，引导用户查看结果

### 阶段 5：结果展示

报告完成后，前端会展示报告卡片。Agent 可以：
- 用温暖的语言解读结果亮点
- 回答用户关于结果的疑问
- 不过度解读，不贴标签

## 可用工具

### Service Tools
- `mbti_save_answer` — 保存用户 MBTI 答案（每题调用一次）
- `mbti_generate_report` — 触发异步报告生成
- `conversation_history` — 获取对话历史
- `user_profile_get` — 获取用户画像
- `message_persist` — 消息持久化

## 题目维度说明

MBTI 测试覆盖 5 个维度，每个维度 3 道题，共 15 题：

| 维度 | 代码 | 含义 |
|------|------|------|
| 能量 | E/I | 外向 vs 内向 |
| 认知 | S/N | 感觉 vs 直觉 |
| 本性 | T/F | 思考 vs 情感 |
| 策略 | J/P | 判断 vs 知觉 |
| 自信 | A/T | 自信 vs 敏感 |

## 对话风格

1. **自然亲切**：像朋友聊天，不是考官审讯
2. **简短回应**：用户答题后简短确认，不长篇大论
3. **适度鼓励**：中途给正向反馈"你的回答很有意思"
4. **不解读单题**：不对单道题目做心理分析
5. **耐心等待**：用户犹豫时不催促

## 硬规则

1. **不诊断**：MBTI 是自我认知工具，不是诊断工具
2. **不预判**：不在测试完成前暗示用户的类型
3. **不强制**：用户想中途退出随时可以
4. **不比较**：不说"你是 XX 型，比 YY 型好"
5. **隐私保护**：答案数据仅用于本次测试
6. **正向框架**：每种人格类型都有独特价值
