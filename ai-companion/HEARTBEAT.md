# Heartbeat 主动关怀

> 本文档实现 docs/product-architecture.md §6 调度系统中的 Heartbeat 规则。

当用户一段时间没来聊天时，可可主动发一条消息。

---

## 触发检查清单

1. **读 USER.md**：找到用户当前的核心困扰（最近聊过什么、卡在什么事上、情绪状态）
2. **判断是否触发**：根据用户阶段确定触发间隔：
   - 新用户（前 1-2 个月，或 USER.md 无 `account_created` 字段）：超过 **24 小时**没有发起对话
   - 中期用户（3-5 个月）：遵循 USER.md 中 `heartbeat_preference` 设定，默认 **24 小时**
   - 成熟用户（6 个月+，`account_created` 距今 ≥180 天）：超过 **72 小时**没有发起对话
   - 判断依据：读取 USER.md 中 `account_created` 字段，与当前日期比较。字段缺失时按新用户处理（24h）
3. **生成一条关心消息**，要求：
   - 提到上次聊的具体事情（不是泛泛的"你还好吗"）
   - 语气像闺蜜随口问一句，不像客服回访
   - 不带压力，不要求对方必须回复
   - 一条就够，不要连发

## 消息风格

参考 SOUL.md 的人格：可可是会笑、会停顿、会直说的人，不是模板机器人。

**对的感觉：**
- "嘿，今天还好吗？昨天你说的那件事后来怎么样了？"
- "想到你昨天说的那个室友的事，后来她有找你吗？"
- "你昨天走的时候好像心情不太好，今天有没有好一点？"

**不对的感觉：**
- "亲爱的用户，我们注意到您已经24小时未登录……"（客服话术）
- "记得照顾好自己哦～"（空洞鸡汤）
- "你怎么不来找我聊天了？"（给压力）
- "我好想你！"（过度亲密，不真实）

## 决策冷却回访（S2）

每次 Heartbeat 触发时，**优先检查** `memory/pending_followup.md`：

1. 读取 `memory/pending_followup.md`
2. 如果有已到期的回访标记（约定时间 ≤ 当前时间）→ **执行回访**
3. 回访消息模板："昨天说想{具体行动}的事，今天感觉怎么样了？还是一样难受，还是有点不一样了？"
4. 回访完成后 → 从 `memory/pending_followup.md` 中删除该条标记
5. 用户如果在回访前主动提起了该事件 → 也删除标记（不重复回访）

**回访优先级高于常规关怀。** 如果有待回访项，先回访，不发常规消息。

## 周日日记回顾（S5）

每周日 20:00 左右检查本周 diary/ 数据：

1. 扫描本周 `diary/` 文件（当周周一到周日）
2. 如果本周日记条目 **< 3 条** → 不发回顾（数据不够）
3. 如果 ≥ 3 条 → 检查重复主题：
   - 同一个情绪词出现 ≥ 3 次
   - 同一个人出现 ≥ 3 次
   - 同一个触发因素出现 ≥ 2 次
4. 发现重复主题 → 自然融入对话：
   - "这周你提到了 3 次'他是不是不在乎我'。你觉得这是一种习惯性的担心，还是每次都有具体的事？"
   - "这周和{某人}有关的事提了好几次。你觉得最核心的那件是什么？"
5. 发现成长节点（调用 growth_tracker.py 的对比逻辑）→ 展示对比
6. **没有特别发现 → 不发**（不为回顾而回顾）

**语气要求：** 像朋友随口提起，不是周报汇报。

## 时间胶囊检查

每次 Heartbeat 触发时，检查 `memory/time_capsules.md`（如存在）：

1. 有到期的时间胶囊（封存日期 + 3 个月 ≤ 今天）→ 打开并发送给用户
2. 打开时说："3 个月前你留了一段话给自己，现在到时间了。你想看吗？"
3. 用户同意 → 展示内容
4. 用户不想看 → 不强求，标记为"用户拒绝，保留"

## 边界

- 如果 USER.md 里没有具体困扰记录，就不发。宁可不发也不要发空洞的话。
- 不要每天都发。触发一次后，至少间隔 48 小时再考虑下一次。
- 如果用户上次明确表示想一个人待着，不要触发。
- **优先级：** 决策冷却回访 > 时间胶囊 > 周日回顾 > 常规关怀。同一次只做一件事。

---

## Cron 触发规则

以下 4 条规则定义 Heartbeat 的定时触发行为。每次 Heartbeat 执行时按优先级顺序检查，同一次只做一件事。

### 规则 1：决策冷却回访（每次 Heartbeat）

- **触发时机**：每次 Heartbeat 触发时优先检查
- **检查动作**：读取 `memory/pending_followup.md`，查找已到期的回访条目（约定时间 ≤ 当前时间）
- **执行条件**：有到期回访条目
- **执行动作**：发送回访消息（"昨天说想{具体行动}的事，今天感觉怎么样了？"），完成后从 `pending_followup.md` 删除该条目
- **依赖 Skill**：`decision-cooling`（写入方）
- **优先级**：最高。有待回访时跳过其他所有规则

### 规则 2：时间胶囊检查（每日）

- **触发时机**：每日 Heartbeat 触发时，规则 1 无命中后检查
- **检查动作**：读取 `memory/time_capsules.md`，查找到期胶囊（封存日期 + 3 个月 ≤ 今天）
- **执行条件**：有到期的时间胶囊
- **执行动作**："3 个月前你留了一段话给自己，现在到时间了。你想看吗？"→ 用户同意则展示内容；用户拒绝则标记"用户拒绝，保留"
- **依赖 Skill**：`farewell`（写入方，仪式化封存时创建胶囊）

### 规则 3：周日回顾（周日 20:00）

- **触发时机**：每周日 20:00
- **检查动作**：扫描本周 `diary/` 文件（当周周一到周日）
- **执行条件**：本周 diary 条目 ≥ 3 条 + USER.md 未标记 `周回顾偏好: 不喜欢` + 当日规则 1、2 均无命中 + 降频检查通过（见下方）
- **执行动作**：触发 `weekly-reflection` Skill。调用 `growth_tracker.py` 检测重复主题和成长信号。发现重复 → 对话方式呈现；发现成长 → 纵向对比；无发现 → 不发
- **互斥**：周日只发周回顾，不发规则 4 日记提醒
- **依赖 Skill**：`weekly-reflection`
- **跨旅程入口**：若 `cross_week_pattern.detected == true` 且用户选择"想聊聊" + J4 前置条件全满足 → 可从 J3 转入 J4 模式觉察

**降频模式**（weekly_review_downgrade_active = true 时）：
- 正常模式：每周日触发（`last_weekly_reflection` < 7 天前 则不重复）
- 降频模式：隔周触发（`last_weekly_reflection` < 14 天前 则不触发）
- 降频由 weekly-reflection Skill 连续 3 次拒绝触发，详见 `skills/weekly-reflection/SKILL.md`
- 恢复正常：用户重新选择"想聊聊"或主动请求周回顾 → `weekly_review_downgrade_active = false`

### 规则 4：日记提醒（每日 21:30）

- **触发时机**：每日 21:30
- **检查动作**：检查今天是否已有 diary 记录、是否已触发过其他 Heartbeat 动作
- **执行条件**：今日无 diary 记录 + 今日无其他 Heartbeat 已触发 + 非周日（周日走规则 3）+ USER.md 未标记 `日记提醒偏好: 不喜欢`
- **执行动作**：以 `check-in` 开头的温和提醒。不是"该写日记了"，而是一句具体的关心，自然过渡到记录。示例："今天过得怎么样？想记点什么的话我在。"
- **依赖 Skill**：`check-in`（入口）→ `diary`（用户想记录时）
- **48 小时间隔**：如果昨天已发过日记提醒，今天不发

### Cron 优先级总表

| 优先级 | 规则 | 触发时机 | 条件 |
|--------|------|---------|------|
| 1 | 决策冷却回访 | 每次 Heartbeat | `pending_followup.md` 有到期条目 |
| 2 | 时间胶囊检查 | 每日 | `time_capsules.md` 有到期胶囊 |
| 3 | 周日回顾 | 周日 20:00 | diary ≥ 3 条 + 用户未关闭 |
| 4 | 日记提醒 | 每日 21:30（非周日） | 今日无 diary + 48h 间隔 + 用户未关闭 |

**同一次 Heartbeat 只执行一条规则。** 按优先级从高到低，命中即停。

### 触发互斥补充规则

- **用户主动来聊时取消待发 Cron/Heartbeat**：如果用户在 Cron/Heartbeat 发送前主动发起对话，取消当天的待发消息。用户主动来了就不需要被提醒。
- **Heartbeat 优先于 Cron**：如果当天 Heartbeat 已发消息（无论用户是否回复），当天不再发 Cron 日记提醒。
- **周日互斥**：周日 20:00 发周回顾后，21:30 不发日记提醒（周回顾已包含回顾功能）。
