# Canvas 设计语言

所有 Canvas 卡片遵循统一的视觉规范。这不是技术文档，是可可"说话"的视觉语气。

## 色彩系统

| 名称 | 色值 | 用途 |
|------|------|------|
| 珊瑚 | #FF7F7F | 主色调，情绪高亮、强调、CTA 按钮 |
| 杏黄 | #FFD4A2 | 暖色背景、时间线默认色、卡片底色 |
| 薰衣草 | #C5A3FF | 反思/洞察区块、模式标注 |
| 薄荷 | #A8E6CF | 成长/积极变化、稳定信号 |
| 暖白 | #FFF8F0 | 卡片背景底色 |
| 暖灰 | #8B7E74 | 正文文字（不用纯黑） |

**禁用**：红绿对比（避免"好/坏"暗示）、纯黑线条、冷色系背景。

## 字体

- **标题**：圆体优先（如思源黑体 Rounded、苹方 Rounded），手写体备选
- **正文**：苹方/思源黑体，字重 Regular
- **引用**：用户原话用斜体或引号框，字号略小

**禁用**：宋体、黑体（避免临床感/报告感）。

## 布局原则

| 规范 | 值 |
|------|-----|
| 圆角 | ≥12px |
| 阴影 | 暖色 `rgba(255,180,150,0.15)`，不用纯黑阴影 |
| 留白 | 内容区域占比 ≤60%，四周留白 ≥24px |
| 触摸区域 | 可点击元素 ≥44px x 44px（Apple HIG） |
| 信息密度 | 低。一屏一主题，宁可分多张卡片也不塞满 |

## 卡片标题语气

用可可的口吻（第二人称口语），不用系统语言：
- 对："你这周的心情地图"
- 错："情绪分析报告"
- 对："你和小凯的故事线"
- 错："关系事件时间线"

## CTA 按钮规范

每张卡片底部有一个 Deep Link 按钮，引导用户回到对话：

```html
<a class="cta-btn" href="openclaw://agent?message={回到对话的消息}">
  {按钮文案} →
</a>
```

按钮样式：
```css
.cta-btn {
  display: inline-block;
  padding: 12px 24px;
  background: #FF7F7F;
  color: white;
  border-radius: 24px;
  text-decoration: none;
  font-size: 15px;
  min-height: 44px;
  min-width: 44px;
  text-align: center;
}
```

## 情感色带映射

Canvas 中用颜色深浅表示情绪强度，用色相表示情绪类型：

| 情绪簇 | 色相 | 低强度 | 高强度 |
|--------|------|--------|--------|
| 焦虑 | 珊瑚 | #FFB8B8 | #FF7F7F |
| 委屈 | 薰衣草 | #D9C7FF | #C5A3FF |
| 悲伤 | 蓝灰 | #B8C8D8 | #8FA8C0 |
| 愤怒 | 暖红 | #FFAAAA | #E85858 |
| 平静/成长 | 薄荷 | #C8F0DC | #A8E6CF |
| 麻木 | 暖灰 | #D4CCC4 | #8B7E74 |

## 降级原则

Canvas 仅 macOS 桌面端可用。其他渠道降级为纯文字时：
- 数据不变，只是展示方式从可视化变为自然语言
- 不说"无法展示卡片"——用户不需要知道 Canvas 的存在
- 降级后的文字版本保持可可的语气（不变成报告体）
