# 教学文档事实性验证规格书

## 被评估文档
`docs/openclaw-anatomy.md` — 小龙虾的六个器官

## 前提
教学质量（Kirkpatrick 4D × 6 器官）已全部通过 ≥ 8。本轮专注于事实性验证与引用标注。

## Features

### Feature 1: 来源研究与下载

**目标：** 找到文档中所有事实性断言的原始来源，下载到 `docs/references/` 目录。

**需要追溯的断言类型：**

| 领域 | 文档中的断言示例 | 需找到的来源 |
|------|----------------|-------------|
| Context Engineering | "金发姑娘区间"、"最小高信号"、"只写不可推断内容" | Anthropic 官方文档原文 |
| Context Engineering | "LLM 生成的规则反而降 3%" | ETH Zurich 论文原文 |
| Prompt Engineering | "位置=优先级"、"示例胜过规则清单" | 官方文档或顶级专家实操报告 |
| 心理学 | 5-4-3-2-1 感官着陆法 | 学术来源（教科书/期刊） |
| 心理学 | 循环叹息（physiological sigh） | Stanford/学术研究 |
| 心理学 | 危机干预五步流程 | QPR/ASIST 或同等标准 |
| AI 技术 | "上下文窗口有限"、"AI 默认什么都做" | 技术文档或专家解释 |

**验收标准：**
- 所有 A 类断言（研究结论、最佳实践、心理学技术、危机干预）有对应的本地文件
- 所有 B 类断言（技术断言、方法论）有对应的来源记录
- 每个来源文件包含：标题、作者、日期、原文关键段落
- 文件存放在 `docs/references/` 目录，命名清晰

---

### Feature 2: 引用标注与准确性验证

**目标：** 在教学文档中添加内联引用，验证所有断言的准确性，同时不破坏教学节奏。

**验收标准（5 个维度，全部 ≥ 8）：**

1. **引用覆盖率 ≥ 8：** A 类断言 ≥ 90% 有引用 + B 类 ≥ 60%，原文已下载到本地
2. **心理学准确性 ≥ 8：** ≥ 90% 技术准确，危机干预流程符合行业标准
3. **AI 领域准确性 ≥ 8：** ≥ 90% 准确，简化处不会导致读者错误操作
4. **L1 反应 ≥ 8：** 读完仍觉得"我完全可以自己做一个不同的 AI"
5. **L3 行为 ≥ 8：** 仍能创建文档没教过的同类产物

**引用格式要求：**
- 内联标注，不打断阅读节奏（如括号内注明来源名）
- 不要变成学术论文——引用服务于教学，不是替代教学
- 首次出现时完整标注，后续出现可简写
