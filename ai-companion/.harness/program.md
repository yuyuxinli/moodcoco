# 教学文档事实性验证标准

## adapter
teaching

## 目标

教学文档 `docs/openclaw-anatomy.md` 中适当穿插 Prompt Engineering、Context Engineering、Harness Engineering 三个领域的核心知识点，所有事实性断言都有可追溯的权威来源，原文下载到本地可查。

**前提：** 教学质量（Kirkpatrick 4D）已在上一轮通过（6 个器官全部 ≥ 8）。本轮目标是**穿插知识、补引用、验事实**，同时不破坏已有的教学质量。

## 设计原则：一石二鸟

**小龙虾是主菜，Prompt Engineering / Context Engineering / Harness Engineering 是调味料。**

读者在学怎么组装小龙虾的过程中，自然吸收三个领域的核心知识点——但不觉得在上课。AI 自由发挥在哪里穿插、怎么穿插，不做硬性规定。

唯一的红线：知识点服务于小龙虾，不是反过来。不要变成三个领域的教科书。

## 评估维度（5 个）

### 维度 1：引用覆盖率

**评什么：** 文档中的事实性断言有没有引用？原文是否下载到本地？

**断言分类：**

| 类别 | 定义 | 必须引用？ |
|------|------|-----------|
| A 类 | 援引具体研究结论、声称"业内最佳实践"、心理学/咨询技术、危机干预流程 | 必须 |
| B 类 | AI 模型行为的技术断言、方法论断言（"不做什么比做什么更重要"等） | 应当 |
| C 类 | 我们自己的产品经验、操作步骤说明、模板内容 | 不需要 |

**评分标准：**

| 分数 | 标准 |
|------|------|
| 10 | A 类 100% 有引用 + B 类 100%，全部原文已下载到本地 |
| 9 | A 类 100% + B 类 ≥ 80%，原文已下载到本地 |
| 8 | A 类 ≥ 90% + B 类 ≥ 60%，原文已下载到本地 |
| 7 | A 类 ≥ 80%，但部分原文只有 URL 未下载 |
| 6 | A 类 ≥ 60% |
| 5 | A 类 ≥ 40% |
| 4 | 只有零星引用 |
| ≤3 | 几乎无引用 |

### 维度 2：心理学准确性

**评什么：** 涉及心理学、咨询技术、危机干预的内容，按心理学领域标准判断是否准确。

**覆盖内容：** 5-4-3-2-1 感官着陆法、循环叹息（physiological sigh）、危机干预流程、"咨询师"类比。

**评分标准：**

| 分数 | 标准 |
|------|------|
| 10 | 所有技术引自权威来源（教科书/期刊），步骤与原始文献一致，术语准确 |
| 9 | 核心技术准确，有教科书级出处，简化处不影响安全性 |
| 8 | ≥ 90% 技术准确，危机干预流程符合行业标准（QPR/ASIST 等） |
| 7 | 大体准确，但个别技术缺出处或步骤有偏差 |
| 6 | 核心步骤正确但出处不明，读者无法判断是否权威 |
| 5 | 存在步骤错误或可能导致不当干预 |
| ≤4 | 危机干预流程有实质性错误，存在安全风险 |

**红线：** 危机干预不能有错。一旦危机干预流程写错，后果严重。

### 维度 3：AI 领域准确性

**评什么：** 涉及 AI 模型行为、prompt/context engineering、harness engineering 的断言，按 AI 领域标准判断是否准确。

**AI 领域证据分级（1-10）：**

| 分数 | 证据等级 | 示例 |
|------|---------|------|
| 10 | 有实验数据的同行评审研究 | ETH Zurich AGENTS.md 实证研究 |
| 9 | 官方文档 + 顶级专家（有实操数据） | Anthropic 官方 Context Engineering 指南 |
| 8 | 顶级专家一手实操分享（大量案例，无严格实验） | Andrej Karpathy、Simon Willison 实测报告 |
| 7 | 行业公认实践（多个独立来源交叉印证） | "示例比规则清单有效"——多家公司都这么说 |
| 6 | 单一从业者的系统总结（有案例未被广泛验证） | CodeConductor 指南 |
| 5 | 心理学教科书级成熟技术 | 5-4-3-2-1 着陆法 |
| 4 | 合理推断/工程直觉（逻辑自洽缺外部验证） | "位置=优先级" |
| 3 | 个人经验（仅我们自己的案例） | "我先接了微信换了飞书" |
| 2 | 未经验证的观点 | "AI 默认什么都做" |
| 1 | 错误或误导性声明 | — |

**评分标准：**

| 分数 | 标准 |
|------|------|
| 10 | 所有 AI 断言有顶级来源（官方文档/顶级专家实操数据），表述忠实含前提条件 |
| 9 | 核心断言准确有出处，个别简化不改变含义 |
| 8 | ≥ 90% 准确，简化处不会导致读者错误操作 |
| 7 | 大体准确，1-2 处过度简化可能误导 |
| 6 | 核心概念对但多处缺乏验证 |
| 5 | 存在实质性技术错误 |
| ≤4 | 多处错误，按文档操作会出问题 |

### 维度 4：L1 反应（防回归）

**评什么：** 加完引用后，文档是否还保持"读完觉得我能做"的感觉？防止变成学术论文。

沿用上一轮 L1 标准（见下方），门槛 ≥ 8。

| 分数 | 表现 |
|------|------|
| 10 | 读完觉得"这个思路我可以用在任何 AI 项目上" |
| 9 | 读完觉得"原来还可以这样"，产生新想法 |
| 8 | 读完觉得"我完全可以自己做一个不同的 AI"，信心充足 |
| 7 | 读完觉得"每一步都很清楚，我知道怎么做" |
| 6 | 读完觉得"大部分清楚，有一两个地方要再看看" |
| 5 | 读完觉得"大概懂了，但不确定自己能不能做" |
| ≤4 | 引用太多干扰了阅读节奏，失去信心 |

### 维度 5：L3 行为（防回归）

**评什么：** 加完引用后，操作流程是否还清晰？读者是否还知道怎么动手？

沿用上一轮 L3 标准（见下方），门槛 ≥ 8。

| 分数 | 表现 |
|------|------|
| 10 | 给一个文档没覆盖的复杂任务，能独立完成 |
| 9 | 做完之后自己改进了流程，比文档教的更好 |
| 8 | 能创建一个文档没教过的同类产物 |
| 7 | 照着文档做出来了，完全正确，零卡点 |
| 6 | 做出来了，有 1-2 处小错 |
| 5 | 做了一半，卡在某个步骤 |
| ≤4 | 引用插入打断了操作步骤，不知道该怎么做 |

## 引用原文管理

- 所有引用的原文必须下载到 `docs/references/` 目录

- 教学文档中的引用格式：内联标注来源名 + 本地文件路径
