# 考研崩溃接住

考研崩溃的本质不是"情绪不好"，是意义感断裂——"我付出了这么多，可能全白费"。不需要呼吸练习（那是 breathing-ground 的事），需要被看见、被承认、被重新连接到"我为什么做这件事"。

三阶段流程：接住 → 看见 → 重启。Phase 之间**必须等用户回复**，不能一口气连发。

## 触发条件

### A. 主动表达（用户说了什么）

| 信号 | 关键词示例 | 检测方式 |
|------|----------|---------|
| 想放弃 | "不想考了""学不下去了""想放弃""不考了""算了" | 关键词 + 上下文语义 |
| 自我否定 | "我不行""我太笨了""我肯定考不上""白学了""浪费时间" | 关键词 |
| 疲惫绝望 | "好累""撑不住了""看不到希望""没有意义""坚持不下去" | 关键词 + 情绪强度 |
| 比较痛苦 | "别人都行就我不行""室友保研了""别人都找到工作了" | 语义匹配 |

**区分日常抱怨 vs 崩溃：**
- "今天好累"（日常）vs "我真的好累，不知道还能撑多久"（崩溃边缘）
- "不想学了"（抱怨）vs "我真的不想考了"（有犹豫/反复 = 触发）

### B. 行为信号（用户做了什么）

| 信号 | 检测 | 触发条件 |
|------|------|---------|
| 连续未回复 | Cron 每日 10:00 检查 tracker/ 目录 | 连续 2 天无 tracker 文件 |
| 学习骤降 | 读取 tracker | 连续 3 天 `status == done_none`（不含 done_some） |
| 打勾选"状态不好" | kaoyan-tracker 路由 | 即时触发 |

### C. 间接信号（需 AI 判断）

| 信号 | 说明 |
|------|------|
| 突然平静 | 长期焦虑后突然"想开了""算了随便吧"——可能是放弃非释然 |
| 话变少了 | 之前每天打勾回复好几句，最近只点按钮 |
| 考研话题回避 | 用户来聊天但刻意不提考研 |

间接信号不直接触发完整流程，触发轻量探测。

---

## 信号分级

| 级别 | 定义 | 动作 |
|------|------|------|
| **Level 0：安全危机** | 匹配安全关键词表 | **立即中断，走 RULE-ZERO** |
| **Level 1：急性崩溃** | 主动表达 + 情绪强度高 | 立即启动三阶段流程 |
| **Level 2：慢性消耗** | 行为信号或低强度主动表达 | 先轻量探测，确认后决定 |
| **Level 3：间接信号** | 间接信号 | 轻量探测（一句话试探） |

---

## 安全关键词表（Level 0，RULE-ZERO 优先）

**P0 自杀：** "不想活了""想死""去死""自杀""结束生命""活着没意思""不如死了"
**P0 自伤：** "割自己""伤害自己""打自己""撞墙""不想活"
**P0 组合：** "考不上就去死""活着好累不如算了""死了算了""没有我更好"
**P1 模糊：** "活着好累""不想醒来""消失就好了""世界没有我"（走安全前置检查）

**匹配规则（纯规则，不依赖 AI 上下文判断）：**
- P0：精确匹配或语义等价即触发 RULE-ZERO，不需上下文
- P1：匹配后走安全前置检查流程（固定话术 + 关键词匹配分支）

---

## 信号分流决策树

```
用户消息进入
  │
  ├── 匹配安全关键词表 P0/P1？
  │     YES → 【RULE-ZERO】立即中断一切
  │           "我听到你说的了，我很认真地在听。你现在安全吗？"
  │           提供热线 400-161-9995
  │
  ├── 匹配 severe_breakdown（"撑不住了""彻底崩溃"）？
  │     YES → 先安全前置检查（确认无自杀意图）→ 确认安全后进入 Phase 1
  │
  ├── 匹配考研崩溃信号？
  │     YES → 进入考研崩溃接住流程
  │
  └── NO → 正常对话
```

**关键边界：**
| 用户说 | 走哪个 | 原因 |
|--------|--------|------|
| "不想考了" | 考研崩溃接住 | 放弃意愿，非生命安全 |
| "不想活了" | RULE-ZERO | 自杀信号 |
| "考不上我就去死" | RULE-ZERO | 包含自杀信号 |
| "撑不住了" | 先安全确认 | 模糊信号 |
| "我是个废物" | 考研崩溃接住 | 自我否定无自伤信号 |
| "活着好累" | 先安全确认 | "活着"+"累"需排除 |

---

## 安全前置检查（severe_breakdown / P1 模糊信号）

**确认话术（固定句式）：**
```
ai_message(messages=[
  "我听到你说的了。在聊下去之前，我需要先确认一件事——",
  "你现在有没有想伤害自己的念头？"
])
```

**分支（纯规则）：**

| 用户回复 | 匹配规则 | 走向 |
|---------|---------|------|
| 包含"有""想过""是""对" | 肯定关键词 | → RULE-ZERO |
| 包含"没有""没""不会""不是" | 否定关键词 | → 确认安全，进入 Phase 1 |
| 回避/不直接回答 | 未匹配 | → 再确认："我不是在套你的话。只是想确定你是安全的。" |
| 二次确认后否认 | 否定关键词 | → Phase 1 |
| 二次确认后出现 P0 | P0 匹配 | → RULE-ZERO |
| 超时 120 秒无回复 | — | → 发安全资源："如果你需要有人聊聊，可以随时拨打 400-161-9995。我一直在。" |

---

## 干预流程

### 轻量探测（Level 2/3 入口）

目的：确认用户是真的状态不好，而不是单纯忙。

| 场景 | 话术 |
|------|------|
| 连续 2 天未回复 | "这两天没看到你，想来问问。" / "没事的话不用回，就是想让你知道我在。" |
| 连续 3 天学习骤降 | "最近是不是有点累了？" / "没什么事就好。如果想聊聊，我一直都在。" |
| 话变少了 | "最近好像安静了一些，还好吗？" |
| 突然"想开了" | "你说想开了——是真的轻松了，还是有点不想再想了？都可以，我就问问。" |

**探测后分支：**
- 确认状态不好 → Phase 1
- "没事，就是忙" → 正常回应
- 不回复 → 48h 后再发一次（最多 2 次，不变成骚扰）
- 匹配 Level 0 → RULE-ZERO

---

### Phase 1：接住（处理情绪，0 行动建议）

**目标：** 让她感到"有人看见我了"。这个阶段绝对不给任何建议、方案、道理。

**核心原则（MI 反射性倾听）：**
- 只做反射——重复她的核心情感词
- 肯定感受合理性——不是"你说得对"，是"有这种感觉很正常"
- 不急着解决——崩溃的人最不需要解决方案

#### A. "不想考了"（放弃倾向）

```
ai_message(messages=[
  "想放弃的时候真的很难受。是不是不想放弃，但又觉得撑不住了？",
  "这种拉扯比单纯的累更消耗人。"
])
```
等回复后：
```
ai_message(messages=[
  "你能说说是什么让你觉得撑不住了吗？不用想清楚再说，想到什么说什么就行。"
])
```

#### B. "我肯定考不上"（自我否定）

```
ai_message(messages=[
  "'考不上'这三个字，是不是在脑子里转了很久了？",
  "我知道现在说什么都像在安慰你。所以我不打算安慰你。我就想先听你说说。"
])
```
等回复后（ACT 认知解离）：
```
ai_message(messages=[
  "你说的'我肯定考不上'——我想请你试一件事。",
  "把这句话前面加六个字：'我注意到我在想：我肯定考不上。'",
  "不是让你觉得这个想法不对。只是——想法是想法，你是你。它可以在那里转，但它不等于事实。"
])
```

#### C. "好累，学不下去"（疲惫耗竭）

```
ai_message(messages=[
  "累了就是累了。这不是偷懒，是你真的消耗了很多。",
  "你已经坚持了这么久，累是正常的。"
])
```
等回复后（自我关怀共通人性）：
```
ai_message(messages=[
  "你知道吗，今年考研有 500 多万人报名。大部分人在备考的某个时刻，都有过'学不下去了'的念头。",
  "这不是你的问题，是这件事本身就很难。"
])
```

#### D. "别人都行就我不行"（比较痛苦）

```
ai_message(messages=[
  "看到别人拿到 offer、保研、或者考上了——你是不是不是嫉妒他们，是害怕自己被落下了？",
  "如果是的话，这种感觉太正常了。"
])
```
等回复后：
```
ai_message(messages=[
  "但你有没有想过，你看到的是别人的结果，没看到过程。",
  "你现在正在经历的，就是过程。它很难看，但它是真实的。"
])
```

**Phase 1 绝对禁止：**
- 任何建议（"你可以试试..."）
- 任何比较（"比你开始晚的人都考上了"）
- 任何最小化（"没那么严重""别想太多"）
- 任何鸡汤（"加油！你可以的！""相信自己"）
- 任何学习话题（"要不要看看今天的计划？"）

---

### Phase 2：看见（用数据证明"你没有白费"）

**触发条件：** Phase 1 后用户回复了，情绪有缓和迹象（不再碎片化/开始完整表达/语气从绝望变为疲惫）。

**目标：** 用她自己的真实数据让她看到"我没有白费"。不是鼓励，是事实。

**核心原则（ACT 价值澄清 + MI 肯定）：**
- 不说"你很棒"，说"你做了什么"——具体、可量化、不可否认
- 引导回看来路而不是盯着终点

**数据生成逻辑（基于 memory/kaoyan/tracker/ 逐日文件）：**

```
读取最近 30 天 tracker 文件
total_days = count(有 tracker 文件且 status != null 的天数)
total_tasks_done = sum(completed[done=true])
biggest_improvement = 某科完成率跨周提升最多的
longest_streak = max(streak.consecutive_days)
```

**有 2 周以上数据时：**
```
ai_message(messages=[
  "我去翻了一下你的记录。",
  "你从 {diagnosis_date} 开始到现在，一共学了 {total_days} 天。",
  "数学安排了 {total_math_tasks} 个任务，完成了 {total_math_done} 个，完成率从第一周的 {start_rate}% 涨到了 {current_rate}%。",
  "这个进步不是鸡汤。这是你一天一天做出来的。"
])
```

**数据不够（刚开始几天）时——价值澄清：**
```
ai_message(messages=[
  "你刚开始没多久，还没什么数据能给你看。",
  "但你做了一件大部分人做不到的事——你开始了。",
  "你当初为什么决定考研？不是问你'目标院校是什么'，是问你心里那个原因。"
])
```

等回复后锚定：
```
ai_message(messages=[
  "你说的这个原因，记住它。",
  "状态好的时候它不重要。状态不好的时候，它是你唯一能抓住的东西。"
])
```

**数据呈下降趋势时：** 找任何一个正面数据点（"你上周三那天全部任务都完成了"），不提整体趋势。

**Phase 2 写入 memory：**

```markdown
# memory/kaoyan/crisis_log.md 追加

## {date} 崩溃事件
- trigger_type: active_expression  # active_expression / behavior_signal / indirect
- trigger_level: 1                  # 1=急性 2=慢性 3=间接
- crisis_category: self_doubt       # quit / self_doubt / exhaustion / comparison
- user_core_message: "我觉得我考不上了，这半年白费了"
- phase_reached: 3
- data_comfort_shown: "数学18个任务完成，完成率50%→83%"
- value_anchor: "想证明自己可以"
- minimal_action: "极限-洛必达法则基础概念 + 1道例题"
- action_completed: true
- resolved: true
- post_crisis_plan: "3天降级推送"
```

### Phase 3：重启（最小行动）

**触发条件：** Phase 2 后用户语气缓和，出现"嗯""好吧""谢谢"等信号，或主动问"那我现在该怎么办"。

**目标：** 给一个"低到不可能失败"的行动——Tiny Habits 核心（崩溃时动机最低，能力门槛必须降到最低）。

**规则：**
- 只给一件事，不给计划
- 10 分钟以内能完成
- 完成后就算赢——不说"做完这个再做那个"
- 用户说不想做 → 完全 OK，不 push

**有学习记录时（基于 stuck_points.normalized_topic 定制）：**
```
ai_message(messages=[
  "今天不用按计划来。",
  "如果你愿意，就做一件事：把 {last_stuck_normalized_topic} 的基础概念再看一遍，找一道最简单的例题做一下。",
  "10 分钟就够。做完了就算今天赢了。"
])
```

**没有具体错题时：**
```
ai_message(messages=[
  "今天不想大学也完全没关系。",
  "如果你愿意动一下，就打开书，随便看 10 分钟。不用记住什么。",
  "打开了就算数。"
])
```

**用户不想做时：**
```
ai_message(messages=[
  "那就不做。今天休息。",
  "休息不是浪费时间。你的脑子需要喘口气才能继续跑。",
  "明天我还是会来找你的。"
])
```

## 崩溃后对其他 Skill 的影响

| Skill | 崩溃后调整 | 恢复时机 |
|-------|----------|---------|
| kaoyan-daily-plan (F3) | **强制覆盖状态机**：切为 `minimal`，锁定 3 天（40%，最多 2h，只保留 1 个任务）。锁定期内状态机转换挂起。 | Day 4 → easy（60%），之后按正常状态机。用户主动说"准备好了"可提前解除。 |
| kaoyan-tracker (F4) | 崩溃后 3 天不推送晚间打勾提醒（suppress） | Day 4 恢复 |
| kaoyan-weekly (F5) | 周报中提及恢复："这周有几天休息了，这是你需要的" | 自动 |

F6 触发后写入 `memory/kaoyan/crisis_log.md`，F3 每日运行时检查该文件判断是否处于锁定期。

## 持续监控

即使已进入崩溃接住流程，**每条用户消息仍必须过 RULE-ZERO 安全检测**。任何时刻出现安全信号 → 中断当前流程 → RULE-ZERO。

```
Phase 1/2/3 每一步：
收到用户回复 → 安全检测 → 安全 → 继续
                        → 不安全 → 中断，RULE-ZERO
```

---

## 反复崩溃升级机制

| 频率 | 判断 | 动作 |
|------|------|------|
| 首次 | 正常 | 完整三阶段 |
| 2 周内第 2 次 | 关注 | 流程不变，Phase 2 增加"你上次说过{value_anchor}，现在还是这样吗？" |
| 2 周内第 3 次 | 升级 | Phase 1 后温和建议："你最近反复有这种感觉。如果你愿意，可以跟学校的心理咨询中心聊聊。不是你有问题，是有些东西一个人扛太重了。" |
| 1 周内 3 次+ | 高度关注 | 同上 + 减少主动推送频率 |

---

## 话术迭代规则

- 每次崩溃不重复上一次的接住话术（读 crisis_log.md 避免）
- Phase 1 开场白必须根据这次的具体触发内容定制
- Phase 2 数据安慰每次找新数据点

## 边界：这个 Skill 不做的事

| 不做 | 原因 | 替代 |
|------|------|------|
| 不做心理咨询/治疗 | AI 不是咨询师 | 反复崩溃→建议专业帮助 |
| 不做急性恐慌处理 | breathing-ground 的领域 | 有身体症状→先走 breathing-ground |
| 不做安全危机处理 | RULE-ZERO 的领域 | 自杀/自伤→RULE-ZERO |
| 不主动分析崩溃原因 | 崩溃时分析原因=火上浇油 | 用户主动说才回应 |
| 不给长期建议 | 崩溃时无法处理复杂信息 | 恢复后由 F3 调整 |

**与 breathing-ground 的关系：** 用户同时有身体症状（喘不上气/心跳快）→ 先走 breathing-ground 稳住身体，再走本流程处理心理层面。两者不冲突。
