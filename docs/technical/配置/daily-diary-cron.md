# 每日情绪日记自动化方案

## 目标

每晚 20:00 自动生成当天的情绪日记草稿，通过微信发送给用户确认，减少用户主动记录的门槛。

---

## OpenClaw Cron Job 配置

```yaml
name: daily-diary-draft
cron: "0 20 * * *"
session: isolated          # 独立 session，不污染主对话上下文
announce: wechat           # 完成后通过微信推送结果给用户
```

### 执行逻辑（伪代码）

```
1. 扫描当天所有对话记录（读取 memory/ 和 diary/ 当日文件）
2. 提取情绪事件（六元组：事件、情绪、强度、想法、触发、应对）
3. 生成草稿 → 写入 diary/YYYY/MM/DD.md（标注为 [草稿]）
4. 通过微信发送草稿摘要，等待用户确认或修改
```

---

## 日记格式

遵循 `skills/diary/SKILL.md`（相对于 workspace 根目录）定义的六元组结构：

```markdown
# YYYY-MM-DD

## HH:MM {一句话标题}

**事件**：{发生了什么}
**情绪**：{精确情绪词}
**强度**：{1-10}/10
**想法**：{当时最突出的想法}
**触发**：{什么引发了这个情绪}
**提到的人**：[{名字}](../../people/{名字}.md)

> "{用户的原话}"

---
```

文件路径：`diary/YYYY/MM/YYYY-MM-DD.md`（追加模式，一天可多条）

---

## 微信推送格式

Cron job 完成后，通过微信发送如下消息：

```
今天的情绪日记草稿好了 ✏️

【20:00 草稿】
{当天最重要的 1-3 条情绪事件摘要}

回复「确认」保存，或直接告诉我要改什么。
```

用户可以：
- 回复「确认」→ 去掉 [草稿] 标注，正式保存
- 回复修改意见 → AI 更新草稿后再次发送
- 不回复 → 草稿保留 7 天后自动归档

---

## 核心约束（来自 diary SKILL）

- **不脑补**：只记录对话中用户实际说过的内容，AI 的推测标注"我观察到"
- **保留原话**：关键表达用引号原文保留
- **第一人称"我"**：日记全部用用户视角的"我"写
- **日期用真实日期**：从系统获取当天日期，不猜

---

## 文件组织

```
diary/
  2026/
    03/
      2026-03-29.md    ← 当天日记（可含多条）
people/
  {名字}.md            ← 自动识别并关联的人物档案
memory/
  2026-03-29.md        ← 跨 session 模式观察（重复情绪/触发时写入）
```
