### 学习工具箱（ljg-skills）

> 来源：`lijigang/ljg-skills`。学习不是独立角色，而是四个角色的共享底座——读完必须有输出，否则就是知识焦虑。

**核心推荐：**

| Skill | 做什么 | 应用场景 |
|-------|--------|---------|
| `/ljg-paper` | 论文阅读器。非学术视角，猎取 idea。Feynman 解释 + PhD 导师审稿 + 三镜头启发（迁移/混搭/反转） | 读心理学论文提取产品洞察、读商业论文找市场信号、读 AI 论文决定技术方案 |
| `/ljg-learn` | 概念解剖刀。8 刀拆解任何概念（锚定/历史/辩证/现象学/语言学/形式化/存在主义/美学），最后压缩为一句话 | 搞懂"Therapeutic Alliance""情绪颗粒度""Context Engineering"等跨领域概念 |
| `/ljg-plain` | 白话引擎。任何内容重写成 12 岁能懂。9 条红线（口语化至上/零术语/短句） | 把 Barrett 情绪构建论变成公众号语言，把 DBT 变成训练营课程语言 |
| `/ljg-rank` | 降秩引擎。把一个领域砍到最少的独立生成元，证明砍法有效 | "情绪消费市场靠什么撑着？""AI 陪伴和真人咨询的本质差异是什么？" |
| `/ljg-writes` | 写作引擎。从一个观点出发，写着写着想明白。1000 字思考短文，反转/追前提/追情绪三把铲子 | 想不清楚的问题先写一篇——"用户付费买的到底是什么？" |

**有用但低频：**

| Skill | 做什么 | 应用场景 |
|-------|--------|---------|
| `/ljg-card` | 内容→PNG 卡片，6 种模式（长图/信息图/多卡/视觉笔记/漫画/白板） | 与 `/wechat-infographic` 互补，非微信场景的可视化用它 |
| `/ljg-invest` | "秩序创造机器"投资分析框架 | 用投资人视角反审自己，或分析竞品 Tolan/Ash |
| `/ljg-roundtable` | 圆桌讨论，请真实历史/当代人物辩论 | 请 Barrett + Ekman 辩论情绪本质，请段永平 + Peter Thiel 辩论商业模式 |
| `/ljg-paper-flow` | paper + card 流水线，读完论文自动出卡片 | 批量读论文时省操作 |

**与已有 Skill 的关系：**

| ljg-skill | 已有 Skill | 关系 |
|-----------|-----------|------|
| `/ljg-card` | `/wechat-infographic` | 互补：ljg-card 通用，wechat-infographic 微信专用 |
| `/ljg-plain` | `/wechat-editorial` | 互补：plain 负责"复杂变简单"，editorial 负责"简单变对" |
| `/ljg-writes` | `/article-blueprint` | 互补：writes 是 1000 字思考，blueprint 是 2000-5000 字公众号长文 |
| `/ljg-paper` | `/eg-emotion-knowledge` | 互补：paper 读新论文，eg-emotion 是已消化的理论库 |
| `/ljg-invest` | `/plan-ceo-review` | 互补：invest 是外部投资人视角，ceo-review 是内部创始人视角 |

### 四帽切换纪律

| 陷阱 | 表现 | 解法 |
|------|------|------|
| CEO 逃逸 | 天天想战略不执行 | CEO 帽子每周最多 2 次，每次 2h |
| 技术黑洞 | 写一个功能写了一周 | 先过 `/epic-hypothesis`，验证需求再动手 |
| PM 完美主义 | 用户故事写了 20 个没开发 | PM 产出 = 假设+验证方案，不是文档 |
| 内容倦怠 | 觉得写公众号没意义想做产品 | 内容是获客引擎，没流量其他都是零 |
| 帽子混穿 | 写代码时突然想改文案 | 时间块：上午内容、下午产品/技术、晚上 CEO |
| 学习上瘾 | 读了 10 篇论文没产出一篇文章 | 学习必须绑定角色输出：读完→写 insight→变成文章/产品决策/技术方案 |
