# 心情可可：待验证需求清单

*2026-03-14*

---

## 核心假设（JTBD）

用户不是在找「AI 聊天」，是在找「凌晨 2 点能接住我的东西」。

产品体验完整链路：**缓下来 → 倒出来 → 被记住 → 理清楚 → 留下来**

---

## 需求 1：记忆

**核心问题：** 记什么？怎么让用户感知到「被记住」？

**已确认：**
- 用户反馈「记忆」是强痛点
- 创始人洞察：不是「AI 记得我」，是「我知道 AI 记得我」（感知层）

**待定：**
- [ ] 记什么：全部记？只记情绪相关？只记关键事件？
- [ ] 怎么让用户感知到：主动提起？展示记忆页面？对话中自然带出？
- [ ] 记忆的呈现形态是什么

**核心假设：**
- A：用户感知到「被记住」后，留存会提升
- B：记忆的内容需要是情绪和关键事件，不是流水账

---

## 需求 2：日记

**核心问题：** 聊完之后留下什么？什么形态？

**已确认：**
- 创始人洞察：AI 的价值 = 倒出来 → 理清楚 → 留下来。缺第三步，用户不会回来

**待定：**
- [ ] 形态：日记？摘要？卡片？时间线？情绪曲线？
- [ ] 自动生成还是用户主动触发？
- [ ] 用户会不会回看？（核心假设 A）
- [ ] AI 整理的结果用户认不认？（核心假设 B）

**核心假设：**
- A：用户聊完之后会想回头看
- B：AI 整理出来的东西，用户觉得「对，这就是我的感受」

---

## 需求 3：场景体验（UI/UX）

**核心问题：** 视频口型同步、音乐、呼吸感的沉浸体验

**已确认：**
- 功能已开发完成

**待定：**
- [ ] 排期上线
- [ ] 与聊天场景怎么衔接（先缓下来再聊？还是并列入口？）
