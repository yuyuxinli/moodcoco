# V3 深度功能校验测试方案

> 2026-03-31 | 基于 11 个并行 Agent 对 product-experience-design.md 7900+ 行规格的逐 Feature 分析

---

## 核心结论

**V2 测了"有没有出错"，V3 要测"有没有做对"。**

V2 的 116+ 项测试本质上是三类：
1. pytest 测脚本函数逻辑（输入→输出）
2. adapter 测关键词/文件存在性（"写进去了没有"）
3. OpenClaw 对话测单轮回复行为（"有没有说错话"）

**V3 要补的是第四类：多轮对话后的文件写入验证（"功能有没有真正生效"）。**

## 共性发现：5 个系统级盲区

### 盲区 1：文件写入验证为零

没有一个测试验证过对话结束后 USER.md / people/*.md / diary/*.md / pattern_log.md 是否被正确写入。所有"记忆"功能的测试止步于脚本函数的输入输出，不看最终文件状态。

**影响**：F01/F04/F05/F06/F07/F08 的核心数据流全部无法确认。

### 盲区 2：多轮对话测试为零

17 条 OpenClaw 对话测试全部是单轮。情绪稳定信号（需多轮累积）、消息缓冲（需连续发送）、旅程转换（需状态变化）、告别仪式（需 5+ 轮）— 全部无法用单轮验证。

**影响**：F04/F05/F07/F08/F10 的旅程流程完全无法确认。

### 盲区 3：Canvas/Poll 触发行为从未验证

5 种 Canvas 卡片只验证了 HTML 文件存在，没验证模板变量是否被填充、数据是否从脚本正确注入、降级路径是否生效。3 种 Poll 只验证了关键词存在，没验证 pollMulti 配置、触发时机、结果写回。

**影响**：F02 的全部核心功能无法确认。

### 盲区 4：Skill 激活精确性从未验证

10 个 Skill 的激活条件（想法 vs 行动区分、前置条件联合检测、互斥规则、频率保护）全部只有关键词存在性检查，没有行为层验证。

**影响**：F03 的全部核心功能无法确认。

### 盲区 5：边缘场景全部是纸面规则

同名消歧 3x3 判定表、记忆冲突 4 级优先级、旅程中断恢复 6 场景、数据故障降级 8 种模式 — 全部写在 AGENTS.md 里但没有运行时验证。

**影响**：F11 的全部功能无法确认。
