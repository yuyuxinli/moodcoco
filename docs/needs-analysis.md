# 心情可可 · 需求分析

## 底层逻辑

**一句话：关系是镜子，用户来看的不是关系，是自己。**

```
关系事件（吵架/已读不回/暧昧/分手）
    ↓
情绪爆发（焦虑/愤怒/委屈/迷茫）
    ↓
来找可可（因为不方便找真人）
    ↓
在对话中看见自己的模式
    ↓
模式被记录下来
    ↓
下一次事件时，可可说"上次也是这样，但这次你先停下来了"
    ↓
用户看见自己的变化
```

三层不是并列选择，是一条线：
- **入口**："不方便跟人聊" → 用户为什么来
- **积累**："关系档案" → 来了之后留下什么
- **价值**："看见自己" → 留下之后得到什么

## 15 场景的共性

### 一个底层 Job
> 在关系引发的混乱情绪中，恢复对自己感受和判断的信任。

### 四个共性 Pain
1. **时间错配**：最需要帮助时（深夜/刚吵完）真人不在
2. **羞耻门槛**：这些焦虑在社交中会被轻视（"才三小时至于吗"）
3. **需要分析但得到结论**：闺蜜说"分了"，但用户需要"帮我理清楚"
4. **知道但做不到**：理智和情绪的撕裂是最核心的痛苦

### 一个共性模式
> 情绪先于策略。用户表面需要"怎么做"，真正需要的是先被安抚才有能力执行。
