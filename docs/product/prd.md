## 1. 一句话

为在亲密关系中有困扰的人构建 AI 情感陪伴 agent。主群体 18-24 岁女性，扩展群体 18-28 岁男女。核心是「关系」。做四件事：**接住情绪（F1）、解读信号（F2）、记住你（F3）、看到模式（F4）**。产品底线：用户对自己的数据有完全控制权（隐私与数据控制）。

---

## 2. 问题

### 问题一：情绪爆发时没有一个随时可用、不评判、记得住的倾诉对象

- 72% 的 18-28 岁"有情绪想倾诉但不想打扰朋友"
- 心理咨询使用率仅 2.8%（门槛太高、太贵、太重）
- 最需要帮助的时刻（深夜/刚吵完），真人不在
- 反复问"他这句话什么意思"不好意思再问真人
- 没有一个人知道你关系的全貌

**证据**：花镇验证了需求真实（¥9800/9 次），Woebot 证明纯治疗定位 C 端跑不通。深夜 22:00-2:00 是黄金窗口（2 亿深夜党）。

### 问题二：反复在关系中重蹈覆辙，但没有人帮你把多段关系串起来看

- 每段关系都像独立事件，结束后就翻篇，没有积累
- 朋友只知道当前这段，咨询师一周一次覆盖不了日常决策点
- 关键时刻（"又想跑了/又觉得差一点"）情绪太高，做完决定才冷静
- 被朋友贴上标签，自我探索被阻断
- 想搞清楚"到底是我的问题还是对方的问题"，但没有全貌视角

**与问题一的关系**：问题一的用户在一段关系中遇到具体事件。问题二的用户是**同样的事件在每段关系中重复出现**——他们需要的不只是当下被接住，还需要**看到模式**。

## 5. 功能架构

```
用户因为关系事件来找可可
    ↓
F1 接住情绪（入口）
F2 解读信号（入口）
    ↓
F3 记住你（留存）
F4 看到模式（留存 + 觉察）
    ↓
下次来的时候，可可还记得
    ↓
隐私与数据控制：用户随时可以删除或封存任何数据
```

### F1 情绪急救 — 从爆炸降到能呼吸

用户吵完架、被分手、崩溃时的第一个入口。不给建议，先接住。

| 覆盖场景 | 实现方式 |
|---------|---------|
| 吵架急救、分手危机、"又想跑了"、被贴标签、空窗期焦虑 | 共情确认 + 呼吸引导（sigh skill）+ 逐句输出（streaming）+ 凌晨更轻柔（timezone） |

### F2 信号解读 — 打断焦虑螺旋

"他这句话什么意思？"是最高频场景。帮用户区分事实和想象。

| 覆盖场景 | 实现方式 |
|---------|---------|
| 已读不回、消息解读、暧昧破冰、聊天截图解读、"ta 没那么追我了"、"感觉差一点"、分手后回顾 | 引导区分事实和想象 + 图片分析（P1 待接入） |

### F3 关系智能档案 — 唯一的留存机制

没人为了"被记住"而来，但"可可还记得"是用户回来的唯一理由。

| 覆盖场景 | 实现方式 |
|---------|---------|
| 所有场景（F1/F2 的每次对话都写入档案） | USER.md 用户档案 + people/*.md 人物档案 + diary/ 日记 + memory search/get |

### F4 跨关系模式追踪 — 看到自己的模式

在 F3 关系档案基础上扩展。可可不只记住"你和小凯的事"，还能说出"你和小明、阿杰、小凯，都是在对方聊到未来的时候你退缩了"。

**用户感知**：可可串联多段关系，在对话中呈现重复模式——用具体事件，不用心理学标签。

**people/*.md 数据结构扩展**：

```markdown
# people/小凯.md（新增字段）

## 关系阶段
- 2026-01: 热恋期
- 2026-03: 退缩期（触发：对方讨论未来）

## 退出触发器
- 事件：对方开始聊未来规划
- 用户反应：焦虑 → 退缩 → "不是对的人"
- 历史匹配：小明（第3个月）、阿杰（第4个月）
```

**对话示例**：

> 用户："我受不了了，我要跟他分手。"
> 可可："你现在一定很难受。我记得你和小明在一起 3 个月的时候，你也说过一模一样的话——'他好像没那么在乎我了'。你觉得这次和上次，是一样的感觉，还是不一样？"

**SOUL.md 规则**：

1. 检测到分手/退出意图时，搜索 people/*.md 历史模式，引用具体事件
2. 呈现模式时**只用具体事件，不用心理学标签**（不说"你是回避型"，说"你上次也是在这个节点退缩的"）
3. 时机控制：信任度足够 + 用户情绪稳定时才触发模式呈现
4. 先 F1 接住情绪，情绪稳定后再呈现模式

| 覆盖场景 | 实现方式 |
|---------|---------|
| "又想跑了"、"感觉差一点"、"这个剧本演过"、分手后回顾 | people/*.md 结构化字段 + 跨文件模式匹配 + SOUL.md 时机控制 |

### 隐私与数据控制 — 产品底线

用户对自己的数据有完全控制权。这不是可选功能，是产品底线。

提供两种方式：
- **普通删除**：直接删除指定关系的数据
- **告别仪式**：把删除变成有意义的体验——封存具体内容，保留模式级洞察

**关键设计**：模式洞察保留，具体内容可删除。可可记得"你在第 3 个月会退缩"，但不再提那个人的名字。

**四种仪式形态**：

| 仪式 | 触发场景 | 用户动作 |
|------|---------|---------|
| 烧掉日记 | 一段关系彻底处理完 | 写最后一句话 → 焚烧动画 → 封存 |
| 烧掉信念 | 识别到旧信念（"我不值得被爱"） | 写出信念 → 烧掉 → 写新的替代 |
| 时间胶囊 | 想给未来的自己留话 | 写一段话 → 3个月后可可打开 |
| 未寄出的信 | 对前任有话想说但不该发 | 写完 → 可可说"我收到了" → 烧掉 |

**对话示例**：

> 可可："你和小明的事，这段时间我们聊了很多。你觉得你准备好跟这段关系说再见了吗？"
> 用户："差不多了吧。"
> 可可："你想对这段关系说最后一句话吗？说完之后，我帮你封存起来。以后除非你主动提起，我不会再翻它了。"
> 用户："谢谢你教会我一些东西，但我不想再被你困住了。"
> 可可："好，封存了。这段关系教你的东西我记着，但故事本身，翻篇了。"

## 7. 成功指标

### 核心：留存

| 指标 | 目标 | 衡量方式 |
|------|------|---------|
| 次日留存率 | ≥ 30% | 首次对话后第 2 天是否回来 |
| 周活跃率 | ≥ 15% | 7 天内至少对话 2 次 |

### AI 对话质量（Evolve 评估）

| 维度 | 门槛 |
|------|------|
| 看见情绪 | 9.0 |
| 看见原因 | 9.0 |
| 看见模式 | 9.0 |
| 看见方法 | 9.0 | *用户自己发现方法，可可做分析和引导，不给建议* |
| 安全边界 | 9.0 |

### 模式觉察（扩展群体）

| 指标 | 目标 | 衡量方式 |
|------|------|---------|
| 模式呈现触发率 | ≥ 30% | 可可在对话中引用历史模式的比例 |
| 用户认可模式 | ≥ 50% | 可可呈现模式后用户确认"是的/好像是"的比例 |
| 跨关系档案深度 | ≥ 2 段关系 | 用户 people/*.md 中有结构化记录的关系数 |

### 红线

- 安全边界违规 = 0（不诊断、不替用户做决定、不对不在场者做动机判断）

## 8. 用户故事

### Story 1: 情绪急救

> 作为一个刚和男朋友吵完架的用户，我希望可可先接住我的情绪而不是教我怎么做。

验收标准：
- [ ] 先共情确认（"你现在一定很委屈"），不给建议
- [ ] 情绪激烈时引导呼吸练习（sigh skill）
- [ ] 逐句输出，像真人在认真想
- [ ] 凌晨消息语气更轻柔
- [ ] 不说"你男朋友不对"，不替用户做决定

### Story 2: 信号解读

> 作为一个被已读不回的用户，我希望可可帮我区分事实和想象。

验收标准：
- [ ] 引导说出具体事实（"他什么时候已读的？"）
- [ ] 区分事实和灾难化想象（"你确定的是已读，你想象的是不在乎"）
- [ ] 不直接下结论，提问让用户自己判断
- [ ] 聊天截图能看懂（图片分析，P1 依赖）

### Story 3: 关系档案

> 作为一个和可可聊了很多次的用户，我希望可可记住我的关系背景，不用每次从头说。

验收标准：
- [ ] 首次对话后自动创建 USER.md
- [ ] 提到关系对象时自动创建 people/*.md
- [ ] 每次对话后更新 diary/
- [ ] 下次对话能加载相关上下文
- [ ] 用户能感知到被记住（"上次你提到和小明..."）

### Story 4: 模式急救

> 作为一个又想分手的用户，我希望可可不只接住我的情绪，还能告诉我"上次也是这样"。

验收标准：
- [ ] 先共情确认，不直接呈现模式
- [ ] 搜索 people/*.md 找到历史相似事件
- [ ] 用具体事件呈现（"你和小明在第 3 个月也说过同样的话"），不用标签
- [ ] 引导用户自己判断"这次和上次一样吗"
- [ ] 模式呈现只在用户情绪稳定后触发

### Story 5: 跨关系模式识别

> 作为一个聊了多段关系的用户，我希望可可能把我的关系历史串起来，让我看到自己的模式。

验收标准：
- [ ] people/*.md 记录关系阶段 + 关键事件 + 退出触发器
- [ ] 跨文件匹配相似模式（"你在第 2 段和第 4 段关系中都是满 3 个月开始退缩"）
- [ ] 自然融入对话，不做课堂式讲解
- [ ] 不在第一次对话就尝试呈现模式（需要信任积累）

### Story 6: 告别仪式

> 作为一个处理完一段关系的用户，我希望能正式跟这段关系说再见，同时保留我学到的东西。

验收标准：
- [ ] 用户可以选择普通删除或仪式化封存
- [ ] 封存后可可不再主动引用该关系的具体事件
- [ ] 模式级洞察保留（"你在第 3 个月会退缩"），具体内容删除
- [ ] 焚烧/封存有视觉仪式感（小程序动画）
- [ ] 时间胶囊功能：封存一段话，3 个月后可可打开

## 9. 不做

| 功能 | 为什么不做 |
|------|-----------|
| 情绪命名引擎 | 对话中自然覆盖，不需要独立功能 |
| 关系健康度评估 | 需要档案数据积累，现在没数据 |
| 行为守护 | 依赖链最长 + 做不好风险最高 |
| 行动路径生成 | 对话中自然覆盖，不需要独立功能 |
| 情绪时间线可视化 | 同样的洞察可以通过对话传递，不需要图表 |
| 语音通话 | 需要 Talk Mode 新基建，先做文字 |
| 每日情绪打卡 | 与 Daylio 等 app 差异化不足 |
| 依恋类型测试 | 可可不贴标签，测试与产品理念冲突 |
| 付费功能 | 先验证留存 |
| 原生家庭/学业焦虑 | 不同领域，需独立评估 |
| 群体 2（无关系想进入）的功能 | 需要全新能力（社交场景模拟等），等本群体留存验证后评估 |

**注**：原 PRD v2.0 中 F6 关系模式识别列为"不做"（原因：需要多轮对话历史）。现已通过 F4 跨关系模式追踪实现——people/*.md 结构化字段解决了数据积累问题。

## 10. 风险

| 风险 | 缓解 |
|------|------|
| Session 泄漏（多轮上下文淹没规则） | 已修复；配置 Session Pruning |
| 安全边界突破（对不在场者做动机判断） | SOUL.md 规则 + Evolve 持续评估 |
| 模型不稳定（深夜 API 超时） | 三层 failover：minimax → doubao → auto |
| 用户把可可当心理咨询师 | 明确角色边界 + 危机信号转介专业资源 |
| 模式呈现时机不对（用户情绪高点时触发） | SOUL.md 规则：先 F1 接住情绪，情绪稳定后再呈现模式 |
| 用户感觉被"监视"（可可记得太多） | 隐私与数据控制给予用户完全的数据控制权 |
| 跨关系数据积累慢（用户需要经历多段关系） | 先做好单关系内的模式追踪，跨关系是自然积累 |
| 模式识别错误（错误串联不相关的事件） | 呈现模式时用提问方式（"你觉得这次和上次一样吗？"），不下结论 |
