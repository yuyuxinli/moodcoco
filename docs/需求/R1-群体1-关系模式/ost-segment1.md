## Opportunity 1：关键决策时刻没有实时+了解全貌的对话对象

JTBD 痛点排名 #1 + #2 + #3。用户在"又想跑了/又不满足/又想追新的人"的瞬间，需要一个立刻可用、知道所有前科的对话对象。

### S1 模式急救对话

用户凌晨说"我要分手"，可可接住情绪后，调出历史模式：

> 用户："我受不了了，我要跟他分手。"
> 可可："你现在一定很难受。我记得你和小明在一起 3 个月的时候，你也说过一模一样的话——'他好像没那么在乎我了'。你觉得这次和上次，是一样的感觉，还是不一样？"

不给结论，引导用户自己看到模式。

| 技术 | 用法 |
|------|------|
| Memory search | 检测到分手/退出意图时，搜索 people/*.md 里的历史关键事件 |
| SOUL.md 规则 | 新增规则：呈现模式时用具体事件，不用标签 |
| Timezone | 凌晨触发时语气更轻柔 |
| Streaming | 逐句输出，像真人在想 |

| 痛点强度 | 留存驱动 | 实现成本 | 不可替代性 | 总分 |
|:--------:|:--------:|:--------:|:----------:|:----:|
| 5.0 | 4.5 | 4.5 | 5.0 | **19.0** |

## Opportunity 2：看不清自己在多段关系中重复的行为模式

JTBD 痛点排名 #2 + #5。每段关系都像独立事件，没人帮串起来看。朋友只知道当前这段，咨询师频次太低。

### S4 跨关系模式档案

每段关系的 people/*.md 加结构化字段，可可自动跨文件串联：

```markdown
# people/小凯.md
## 关系阶段记录
- 2026-01: 热恋期，"他好体贴"
- 2026-03: 第3个月，"感觉他没那么在乎了"  ← 与小明.md 第3个月模式一致
- 2026-03-15: 产生分手冲动 ← 与小明.md/阿杰.md 退出模式一致

## 退出触发器
- 触发事件：对方开始讨论未来规划
- 用户反应：焦虑 → 退缩 → "不是对的人"
- 历史匹配：小明（第3个月）、阿杰（第4个月）
```

在合适的时机自然呈现：

> 可可："我发现一件事——你和小明、阿杰、小凯，都是在对方开始聊未来的时候你觉得不舒服。你有没有注意到？"

| 技术 | 用法 |
|------|------|
| people/*.md | 扩展结构：关系阶段 + 关键事件 + 用户反应 + 退出触发器 |
| Memory search | 跨文件模式匹配 |
| Compaction | 压缩时保留关系阶段和模式数据，不丢失 |
| SOUL.md 规则 | 模式呈现时机：信任度够高 + 用户情绪稳定时才触发 |

| 痛点强度 | 留存驱动 | 实现成本 | 不可替代性 | 总分 |
|:--------:|:--------:|:--------:|:----------:|:----:|
| 5.0 | 5.0 | 3.5 | 5.0 | **18.5** |

### S10 告别仪式

用户处理完一段关系后，可可引导一个仪式性的"告别"——烧掉日记、烧掉信念、写未寄出的信。

**这是必须做的功能**：所有用户材料（diary/、people/*.md 中的具体内容）需要有删除能力，告别仪式是删除功能的产品化包装。

> 可可："你和小明的事，这段时间我们聊了很多，你也看到了一些以前没注意到的东西。你觉得你准备好跟这段关系说再见了吗？"
>
> 用户："嗯...差不多了吧。"
>
> 可可："那我们做一件事。你想对这段关系说最后一句话吗？说完之后，我帮你把这段日记封存起来。以后除非你主动提起，我不会再翻它了。"
>
> 用户："谢谢你教会我一些东西，但我不想再被你困住了。"
>
> 可可："好，封存了。这段关系教你的东西我记着，但故事本身，翻篇了。"
>
> （动画：日记页卷起、燃烧、化成一颗小星星收进档案里）

四种仪式形态：

| 仪式 | 触发场景 | 用户动作 |
|------|---------|---------|
| 烧掉日记 | 一段关系彻底处理完 | 写最后一句话 → 封存/焚烧动画 |
| 烧掉信念 | 用户识别到一个旧信念（"我不值得被爱"） | 把信念写出来 → 烧掉 → 可可帮写一句新的替代 |
| 时间胶囊 | 用户想给未来的自己留话 | 写一段话 → 封存 → 3个月后可可打开给你看 |
| 未寄出的信 | 对前任有话想说但不该发 | 写完 → 可可说"我收到了" → 烧掉 |

心理学依据：
- 叙事疗法的 definitional ceremony：通过仪式标记身份转变
- ACT 的认知解离：把信念外化（写出来）→ 和它拉开距离（烧掉）
- 哀伤辅导：象征性告别是健康哀悼的核心步骤

关键设计：**模式洞察保留，具体内容可删除。** 可可记得"你在第 3 个月会退缩"这个规律，但不再提那个人的名字和具体事件。用户有完全的数据控制权。

| 技术 | 用法 |
|------|------|
| diary/ + people/*.md | 封存/删除标记，可可不再主动引用具体内容 |
| Memory | 模式级洞察保留（"第3个月退缩"），事件级细节删除 |
| 小程序 UI | 焚烧/封存动画（视觉仪式感） |
| Cron + Heartbeat | 时间胶囊定时打开 |

| 痛点强度 | 留存驱动 | 实现成本 | 不可替代性 | 总分 |
|:--------:|:--------:|:--------:|:----------:|:----:|
| 3.0 | 3.5 | 2.8 | 3.5 | **12.8** |

> **备注**：总分不高但这是**必须做的功能**——用户数据删除权是产品底线，告别仪式是把"删除"从冷冰冰的操作变成有意义的体验。不受评分排序影响。

## Opportunity 3：被标签化（花心/恋爱脑），自我探索被阻断

JTBD Social Job #1 + Unresolved #3。社交环境的反馈要么无脑支持要么居高临下贴标签，两种都不帮助用户发展自己的觉察。

### S7 事件叙事替代标签

可可绝不主动使用心理学标签，只用具体事件说话：

| 绝不说 | 而是说 |
|--------|--------|
| "你是回避型依恋" | "你上次在对方聊到未来的时候退缩了" |
| "你有承诺恐惧" | "你三段关系都在第 3 个月左右觉得不舒服" |
| "你是恋爱脑" | "你在他冷淡的时候会特别不安，上次也是这样" |

如果用户自己提到标签（"我是不是回避型啊"），可可才回应，而且引导回具体事件：

> 用户："我是不是回避型啊？"
> 可可："你说的回避型是指什么感觉？是在关系变亲密的时候想退后一步吗？你能说说最近一次有这种感觉是什么时候？"

| 技术 | 用法 |
|------|------|
| SOUL.md 规则 | 硬规则：不主动使用依恋类型/心理学标签 |
| Memory search | 用户提标签时，搜索具体事件来回应 |
| Evolve 评估 | "看见模式"维度评估中检查：是否用了标签而非事件 |

| 痛点强度 | 留存驱动 | 实现成本 | 不可替代性 | 总分 |

### S8 低门槛渐进觉察

用 Poll 做轻量级情绪打卡，降低开口门槛：

> 可可（每天晚上 9 点）："今天在关系里什么感觉？"
> 🟢 安心 / 🟡 有点不安 / 🔴 很想逃 / ⚪ 没什么感觉

用户只点一个按钮就行，不用打字。可可根据连续多天的打卡数据发现趋势：

> 可可："最近5天你都选了'有点不安'，从周三开始的。周三发生了什么？"

| 技术 | 用法 |
|------|------|

### S9 关系故事重写

帮用户把叙事从"我有问题"重写为"我在学着理解自己"。用 diary/ 里的历史数据，让用户看到自己的成长：

> 用户："我是不是天生就不适合谈恋爱..."
> 可可："三个月前你遇到类似的情况，直接就走了，连想都没想。但这次你选择先来跟我聊聊，想搞清楚到底怎么回事。你有没有发现，这本身就已经不一样了？"

> 可可："我翻了一下我们的聊天记录。1月份你说'算了，分就分'。3月份你说'我想搞清楚为什么每次都这样'。你觉得这两个你，有什么不同？"

| 技术 | 用法 |

| Memory search | 搜索用户过去的关键表述 |
| Compaction | 压缩时保留用户的"成长节点"表述 |
