## 一、目标客户 & 使用情境

**目标客户：** 18-25 岁女大学生/研究生，高敏感、容易情绪内耗

**Job 发生的情境（"当……的时候"）：**

| # | 情境 | 来源 |
|---|------|------|
| S1 | 深夜睡不着，脑子里的念头停不下来 | 访谈 |
| S2 | 和男朋友吵完架/暧昧对象不回消息 | 访谈 |
| S3 | 考研/论文 DDL 逼近，焦虑到什么都做不了 | 访谈 |
| S4 | 放假回家，和父母起冲突 | product-marketing-context |
| S5 | 刷到别人的好生活，觉得自己很差 | [假设——待验证] |

**当前替代方案（用户今天"雇佣"的东西）：**

| 替代方案 | 她怎么用 | 哪里不够好 |
|---------|---------|-----------|
| 朋友/闺蜜 | 发微信语音吐槽 | 不能随时打扰；太私密的话说不出口；怕被评判 |
| 备忘录/朋友圈仅自己可见 | 写下来发泄 | 没有回应，写完更孤独；写完就忘了 |
| 网易云评论区/树洞 App | 匿名倾诉 | 没人接住你；说完就沉了；没有后续 |
| 豆包/DeepSeek | 随便聊聊 | 套路感；聊完就忘；没有连续性；不懂情绪 |
| 算命/塔罗/测测 | 求"确定性" | 本质是要一个答案来安抚焦虑，不是真信 |
| 心理咨询 | 约咨询师面谈 | 太贵（300-800/次）；预约麻烦；"我没病到那个程度" |
| 什么都不做，硬扛 | 等情绪自己过去 | 内耗加重，影响睡眠/学业/社交 |

## 二、Customer Jobs

### 功能性任务（Functional Jobs）

> 她需要完成什么？

| # | Job | 证据强度 | 来源 |
|---|-----|---------|------|
| F1 | **把脑子里乱七八糟的情绪倒出来**——不用组织语言，多乱都行 | ★★★ | 80% 访谈用户 + 创始人体验 |
| F2 | **理清楚"我到底怎么了"**——从混乱中看到一个说得通的脉络 | ★★☆ | 创始人洞察2，待用户验证 |
| F3 | **留下一个能回看的东西**——不是聊完就没了，而是有迹可循 | ★★☆ | 创始人洞察2，待用户验证 |
| F4 | 在特定痛苦时刻快速让自己好一点（不是彻底解决，是"先活过今晚"） | ★★☆ | 访谈线索：失恋/考研焦虑时找 AI |
| F5 | 记录自己的情绪变化，看到"我在变好" | ★☆☆ | [假设——待验证] |

### 社会性任务（Social Jobs）

> 她希望在别人眼中是什么样？

| # | Job | 证据强度 | 来源 |
|---|-----|---------|------|
| S1 | **不想被看到"我需要帮助"**——用 AI 而不是找人，就是因为不想暴露脆弱 | ★★★ | "不想麻烦别人"（80% 访谈） |
| S2 | 希望被认为是"情绪稳定的人"，不想总是找朋友倒苦水 | ★★☆ | 访谈推断 |
| S3 | 在社交中表现得体——不想因为情绪失控说错话、做错事 | ★☆☆ | [假设——待验证] |

### 情感性任务（Emotional Jobs）

> 她想要什么感受？想逃避什么感受？

| # | 想要的 | 想逃避的 | 证据强度 |
|---|-------|---------|---------|
| E1 | **被看见、被记住**——"有人知道我经历了什么" | 被忽视、说了也没人在意 | ★★★（创始人洞察1 + 访谈） |
| E2 | **被接住**——说完之后有回应，不是对着空气说话 | 说完更孤独（树洞/备忘录的问题） | ★★☆ |
| E3 | 感到"我不是唯一一个这样的人" | "只有我这么矫情" 的羞耻感 | ★★☆（访谈："不是只有我这样"） |
| E4 | 感到掌控感——"我知道我怎么了，我可以做点什么" | 无助、失控、"脑子停不下来" | ★★☆（创始人体验） |
| E5 | 安全感——这里不评判我，不会泄露我说的话 | 被评判、被传出去 | ★★☆（访谈推断） |

## 三、Pains（痛点）

### 障碍（Challenges）

| # | 痛点 | 严重程度 | 说明 |
|---|------|---------|------|
| P1 | **凌晨 2 点最难受，但没人可以找** | 🔴 急性 | 情绪高峰往往在深夜，闺蜜在睡觉，咨询师不上班 |
| P2 | **想说但说不出口**——觉得"太矫情""没什么大不了" | 🔴 急性 | 自我否定阻止了求助行为 |
| P3 | **说完之后什么都没留下**——聊完关掉，下次还是从头来 | 🟡 慢性 | 没有积累，情绪管理没有进步感 |
| P4 | 通用 AI 的回复太套路——"我理解你的感受""你可以试试深呼吸" | 🟡 慢性 | 用户能感知到模板回复，信任崩塌 |
| P5 | 心理咨询门槛太高——价格、预约、"我没病到那个程度"的心理障碍 | 🟡 慢性 | 轻度需求被挡在正式服务之外 |

### 代价（Costliness）

| # | 代价 | 类型 |
|---|------|------|
| C1 | 情绪内耗导致失眠 → 第二天上课/工作状态差 | 时间 + 健康 |
| C2 | 反复找朋友倒苦水 → 社交关系消耗 | 关系 |
| C3 | 心理咨询 300-800 元/次，学生承受不起 | 金钱 |
| C4 | 自己硬扛 → 小问题拖成大问题（焦虑→抑郁） | 健康风险 |

### 现有方案未解决的问题（Unresolved Problems）

| # | 问题 |
|---|------|
| U1 | **没有一个东西同时做到"随时可用 + 有回应 + 记得我"** |
| U2 | 倾诉完之后没有"整理"步骤——情绪发泄了但认知没提升 |
| U3 | 没有从"这次难受"到"下次遇到类似情况我可以怎么办"的桥梁 |
| U4 | 隐私焦虑——和真人说怕被传，和 AI 说怕数据泄露 |

## 四、Gains（期望收益）

### 超出预期（Expectations）

| # | 如果做到这个，用户会惊喜 |
|---|------------------------|
| G1 | "它居然记得我上次说过什么"——不用每次都重新解释背景 |
| G2 | "它帮我理清楚了"——我说了一堆乱的，它帮我看到一条线 |
| G3 | "我能看到自己的变化"——回看一个月前的情绪记录，发现自己好了很多 |

### 节省（Savings）

| # | 节省了什么 |
|---|----------|
| G4 | 不用纠结"要不要打扰闺蜜"——直接打开就说 |
| G5 | 不用花 300 块去咨询师那里——轻度需求这里就够了 |
| G6 | 不用自己从混乱中硬想——AI 帮你整理 |

### 促进采用的因素（Adoption Factors）

| # | 什么会让她愿意试？ |
|---|------------------|
| G7 | 微信小程序，不用下载 App——打开就能用 |
| G8 | 免费就能聊——零门槛，不需要决策成本 |
| G9 | 看到"和我一样的人在用"——小红书上有同龄人的真实分享 |
| G10 | 隐私安全——不需要实名，不会被人发现 |

### 生活改善（Life Improvement）

| # | 如果这个 Job 变容易了，她的生活会怎样 |
|---|-------------------------------------|
| G11 | 深夜难受的时候有个出口，第二天能正常上课 |
| G12 | 慢慢学会"我怎么了"→"我可以怎么办"，内耗减少 |
| G13 | 不再因为"找不到人说"而硬扛，小问题不会拖成大问题 |

## 五、优先级排序

### 最高强度痛点（去小红书验证这些）

| 排名 | 痛点 | 为什么排最高 |
|------|------|-------------|
| 1 | **P1 凌晨没人可找** | 急性、高频、现有方案完全空白 |
| 2 | **P2 想说但说不出口** | 阻止了所有求助行为，是"沉默的大多数" |
| 3 | **P3 说完什么都没留下** | 创始人核心洞察，如果验证通过是差异化关键 |

### 最关键的 Job（产品必须做到的）

| 排名 | Job | 为什么 |
|------|-----|--------|
| 1 | **F1 把情绪倒出来** | 入口行为，用户的第一个动作，做不到后面都没意义 |
| 2 | **E1 被看见、被记住** | 从"用一次"到"回来用"的分水岭 |
| 3 | **F2 理清楚** | 从"发泄工具"到"有价值的产品"的跃迁 |

## 七、对产品的启示

### 如果 JTBD 成立，心情可可应该是：

```
用户在 [深夜/吵完架/焦虑发作] 的时候，
打开心情可可，
把脑子里的混乱全部倒出来（F1），
可可记得她上次说过什么（E1），
帮她把混乱理成一条看得懂的线（F2），
留下一个她明天能回看的东西（F3），
让她觉得"有人知道我经历了什么"（E1），
而不需要打扰任何人（S1）。
```

### 最核心的一句话

**用户不是在找"AI 聊天"，是在找"凌晨 2 点能接住我的东西"。**
