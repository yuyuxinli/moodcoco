# 心情可可：待验证需求清单

*2026-03-14*

---

## 核心假设（JTBD）

用户不是在找「AI 聊天」，是在找「凌晨 2 点能接住我的东西」。

产品体验完整链路：**缓下来 → 倒出来 → 被记住 → 理清楚 → 留下来**

---

## 需求 1：记忆

**核心问题：** 记什么？怎么让用户感知到「被记住」？

**已确认：**
- 用户反馈「记忆」是强痛点
- 创始人洞察：不是「AI 记得我」，是「我知道 AI 记得我」（感知层）

**待定：**
- [ ] 记什么：全部记？只记情绪相关？只记关键事件？
- [ ] 怎么让用户感知到：主动提起？展示记忆页面？对话中自然带出？
- [ ] 记忆的呈现形态是什么

**核心假设：**
- A：用户感知到「被记住」后，留存会提升
- B：记忆的内容需要是情绪和关键事件，不是流水账

---

## 需求 2：日记

**核心问题：** 聊完之后留下什么？什么形态？

**已确认：**
- 创始人洞察：AI 的价值 = 倒出来 → 理清楚 → 留下来。缺第三步，用户不会回来

**待定：**
- [ ] 形态：日记？摘要？卡片？时间线？情绪曲线？
- [ ] 自动生成还是用户主动触发？
- [ ] 用户会不会回看？（核心假设 A）
- [ ] AI 整理的结果用户认不认？（核心假设 B）

**核心假设：**
- A：用户聊完之后会想回头看
- B：AI 整理出来的东西，用户觉得「对，这就是我的感受」

---

## 需求 3：场景体验（UI/UX）

**核心问题：** 视频口型同步、音乐、呼吸感的沉浸体验

**已确认：**
- 功能已开发完成

**待定：**
- [ ] 排期上线
- [ ] 与聊天场景怎么衔接（先缓下来再聊？还是并列入口？）

## 需求 4：趣味心理测试与引导式问答

*来源：同学建议，2026-04-01*

**核心问题：** 用「有趣」作为入口，低门槛收集用户的心理状态和需求，同时让用户在游戏化体验中获得自我认知。

**三个方向：**

### 4A：有趣的心理测试
- 依恋风格测试（焦虑型/回避型/安全型）
- 情绪颗粒度测试（你能区分多少种情绪？）
- 关系模式测试（你在亲密关系里是什么角色？）
- 压力应对风格测试
- 性格优势测试（VIA Character Strengths）

**与竞品的差异化机会：** 测试结果不是贴标签，而是引出一段对话（"你选的这几个，我想多了解一下……"）

### 4B：有趣的心理问题收集
- 冰山问题："你表面上烦的是什么？底下藏着什么？"
- 时光机问题："如果你现在能给一年前的自己发一条消息，你会说什么？"
- 角色互换："如果你最好的朋友遇到了同样的事，你会怎么对 ta 说？"
- 例外问题（SFBT）："有没有哪次，你以为会很崩，但撑过去了？"

### 4C：引导式问答框架（可研究参考）
| 框架 | 核心机制 | 适用场景 |
|------|---------|---------|
| **动机访谈（MI）** | 开放式问题 + 反映性倾听，帮用户自己发现改变动机 | 行为改变、矛盾心理 |
| **焦点解决短期疗法（SFBT）** | 奇迹问题、例外问题、刻度化问题 | 目标导向、找出已有资源 |
| **叙事疗法** | 外化问题、改写故事 | 自我认同、从问题中分离 |
| **Pennebaker 表达性写作** | 连续写 15-20 分钟不停笔，不管语法 | 情绪加工、创伤整合 |
| **ACT 价值观澄清** | "你希望自己的生活是关于什么的？" | 意义感、长期方向 |

**待定：**
- [ ] 测试结果如何与对话打通（测完直接进入 MOOD session？）
- [ ] 趣味测试和正式评估量表（PHQ-9、GAD-7）如何结合？
- [ ] 用户是否愿意"玩测试"——是作为冷启动入口，还是作为日常功能？
- [ ] 数据收集：测试结果是否写入用户档案（people/*.md / USER.md）

**核心假设：**
- A：「有趣」能显著降低第一次开口的心理门槛
- B：测试结果能帮助 coco 更快了解用户，减少首次接触的摸索轮次
- C：用户在「玩」的过程中已经在进行情绪加工，测试本身就是干预
