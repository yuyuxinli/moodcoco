## Opportunity 1："我不够好所以没人要"的核心信念固化

JTBD 痛点排名 #1。这是所有痛点的根源——"我不够好"让用户在社交机会面前自动退缩，任何技巧都被这层信念挡在外面。不触动这个信念，后续所有练习都没有基础。

理论依据：Dweck (2006) 固定型心智模式、Higgins (1987) 自我差异理论、Rogers (1961) 无条件积极关注。

### S1 归因松动对话

用户说"我就是不行"时，可可不正面反驳，而是引导从"我不行"（固定归因）滑向"我还没学会"（成长归因）：

> 用户："我 24 了还没谈过恋爱，是不是真的有问题。"
> 可可："你说'有问题'的时候，你指的是哪种？是觉得自己哪里不好，还是觉得自己跟别人不一样？"
> 用户："就...别人好像都很自然就有了，我好像怎么都不行。"
> 可可："你说的'不行'——你觉得这是你天生就缺了什么东西，还是你没碰到过机会练习？我问这个是因为，你能这么具体地描述自己的感受，说明你的感知力其实很强。不是'不行'，可能只是这件事你还没开始练。"

不给鸡汤，不说"你很好"，只做一件事：把"我不行"（身份判断）拆成"我在这件事上还没经验"（行为描述）。

| 技术 | 用法 |
|------|------|
| SOUL.md 规则 | 检测到"我不行/有问题/不够好"等身份化归因时，引导拆解为行为描述；绝不正面反驳 |
| Memory search | 搜索用户历史中的自我评价表述，追踪"身份归因→行为归因"的变化 |
| Timezone | 深夜触发时语气更温和，不做认知干预，先接住情绪 |
| Streaming | 逐句输出，像在认真想而不是背课文 |

| 痛点强度 | 留存驱动 | 实现成本 | 不可替代性 | 总分 |
|:--------:|:--------:|:--------:|:----------:|:----:|
| 5.0 | 4.0 | 4.5 | 4.2 | **17.7** |

### S2 微证据收集

用户觉得"没有人喜欢过我"时，可可帮他一起找被忽略的微小证据——不是证明"有人喜欢你"，而是证明"你有能力建立连接"：

> 用户："我觉得我从来没被谁喜欢过。"
> 可可："我想问你一件事——不是关于喜欢，是关于连接。你有没有哪个时刻，跟一个人聊着聊着，发现对方比你预期的更愿意继续聊？哪怕是很短的一瞬间。"
> 用户："...好像有一次，一个同学问我作业的事，后来聊了半小时别的。"
> 可可："你注意到了——她本来只需要 2 分钟问完作业，但她选择多聊了 28 分钟。这不是礼貌能解释的。你在那半小时里，让她觉得值得待着。你记得你们聊了什么吗？"

让用户自己发现证据，不是可可"给"他证据。Bandura (1977)：掌控体验是自我效能感最强的来源，这个群体有很多微小的掌控体验，只是从来没注意到。

| 技术 | 用法 |
|------|------|
| SOUL.md 规则 | 不正面反驳"没人喜欢我"，引导用户自己找微证据 |
| diary/ | 每次发现的微证据写入日记，积累成"连接能力档案" |
| Memory search | 跨对话搜索用户提到的正向社交经历，在低落时调出 |
| Heartbeat | 用户连续几天没来时，可可主动提起上次发现的微证据 |

| 痛点强度 | 留存驱动 | 实现成本 | 不可替代性 | 总分 |
|:--------:|:--------:|:--------:|:----------:|:----:|
| 5.0 | 4.5 | 3.8 | 4.5 | **17.8** |

### S3 社交反刍急救

用户从社交场合回来后反复回放自己的"失败"（"我刚才说的那句话好蠢"），可可帮打断反刍螺旋：

> 用户（晚上 11 点）："今天聚会上我跟一个女生说话，说了句超蠢的话，她笑了一下就走了。完蛋了。"
> 可可："你现在脑子里是不是在反复播放那个画面？"
> 用户："对，一直在想。"
> 可可："我问你一件事——她笑的那一下，是嘲笑的笑，还是有可能只是正常的反应？你能确定是哪种吗？"
> 用户："...其实我不确定。"
> 可可："你不确定，但你的大脑已经自动选了最坏的那个版本。你注意到了吗——你的焦虑不是因为她的反应，是因为你自己脑补的那个版本。如果她只是正常地笑了一下然后刚好有事走了呢？"

和 F2 信号解读同一个底层逻辑，但场景从"解读对方消息"变成了"解读自己的社交表现"。帮用户区分"事实"和"脑补"。

| 技术 | 用法 |
|------|------|
| SOUL.md 规则 | 检测到社交后反刍时，引导区分事实和脑补 |
| Timezone | 深夜先接住情绪再做认知拆解 |
| diary/ | 记录反刍事件和认知拆解结果 |
| Streaming | 逐句输出，像在一起想而不是在教他 |

| 痛点强度 | 留存驱动 | 实现成本 | 不可替代性 | 总分 |
|:--------:|:--------:|:--------:|:----------:|:----:|
| 4.5 | 4.8 | 4.5 | 3.8 | **17.6** |

## Opportunity 2：没有安全的练习场——不社交学不会，社交有真实代价

JTBD 痛点排名 #2。鸡生蛋死循环：想学会社交得去社交，但每次社交都有真实代价（被拒绝、丢脸、自信崩塌）。AI 的结构性优势在于提供"零代价试错"。

理论依据：Bandura (1977) 替代经验和情绪唤起管理、Wolpe (1958) 系统脱敏。

### S4 社交场景模拟

可可扮演"喜欢的人"，用户练习真实场景中的对话。关键：不是 PUA 话术训练，而是帮用户找到"符合自己性格的表达方式"：

> 用户："我想练练怎么跟喜欢的女生搭话，但我不知道说什么。"
> 可可："好，我们试试。你先跟我说说，你平时跟朋友聊天什么风格？话多的还是话少的？喜欢开玩笑还是比较认真？"
> 用户："话不多，比较认真那种。"
> 可可："那我们就用你自己的风格来。不用学话多的人。我现在是你的同学，我们刚下课，你想跟我搭话。你会说什么？说什么都行，说错了也没关系——这里只有我们。"
> 用户："呃...那个，你觉得今天老师讲的那个案例有意思吗？"
> 可可（扮演同学）："还行吧，不过我觉得他举的那个例子有点老了。"
> 可可（退出角色）："你刚才做了一件很好的事——你用了一个你真的感兴趣的话题，不是硬编一个搭讪理由。这比任何话术都自然。你感觉怎么样？"

三层设计：
1. **练之前**：先了解用户的社交风格，不给通用模板
2. **练的时候**：可可进入角色扮演，给出真实但安全的反应
3. **练完之后**：可可退出角色，做具体反馈（"你做了什么 + 为什么有效"）

| 技术 | 用法 |
|------|------|
| SOUL.md 规则 | 新增"练习场模式"：进入角色扮演时明确标记，退出时给具体反馈；禁止教话术/套路 |
| USER.md | 记录用户社交风格偏好（内向/外向、幽默/认真等） |
| diary/ | 每次练习记录：尝试了什么 + 反馈 + 感受，追踪进步 |
| Memory search | 调取之前的练习记录 |
| Streaming | 角色扮演时像真人对话节奏 |

| 痛点强度 | 留存驱动 | 实现成本 | 不可替代性 | 总分 |
|:--------:|:--------:|:--------:|:----------:|:----:|
| 5.0 | 5.0 | 3.0 | 5.0 | **18.0** |

### S5 拒绝脱敏训练

用渐进暴露帮用户降低对拒绝的恐惧：

> 可可："你最怕的是什么？是被拒绝本身，还是被拒绝之后那种'完了，我果然不行'的感觉？"
> 用户："后者吧。被拒绝那一下还好，之后那几天一直在想才是最难受的。"
> 可可："那我们做个实验。我现在扮演一个对你没兴趣的人。你跟我搭话，我会礼貌地拒绝你。我们看看拒绝真的发生的时候，你实际的感受是什么——跟你想象中一不一样。准备好了吗？"
> 用户："...行，试试。"
> 可可（扮演）："啊，谢谢你邀请，不过我周末有安排了。"
> 可可（退出角色）："好，刚才那一下，你身体有什么感觉？"
> 用户："心跳快了一下，但其实...好像没有想象中那么可怕。"
> 可可："对。你想象中的拒绝是灾难性的，但真正体验的时候，它就是一句话。你的大脑把'被拒绝'和'我不够好'绑在一起了，但实际上它们是两件事。"

系统脱敏（Wolpe, 1958）：在安全环境中反复暴露焦虑刺激，焦虑反应逐渐降低。关键是暴露后的认知处理。

| 技术 | 用法 |
|------|------|
| SOUL.md 规则 | 练习场模式下的"拒绝场景"子模式 |
| diary/ | 记录每次脱敏的情绪反应强度 |
| Memory search | "上次你说被拒绝'没想象中可怕'，这次呢？" |
| USER.md | 记录拒绝敏感度变化轨迹 |

| 痛点强度 | 留存驱动 | 实现成本 | 不可替代性 | 总分 |
|:--------:|:--------:|:--------:|:----------:|:----:|
| 4.8 | 4.5 | 3.0 | 5.0 | **17.3** |

### S6 信号解读练习

用户不知道对方是友好还是有好感，可可帮建立基本信号解读框架：

> 用户："今天那个女生跟我说'下次一起吃饭吧'，是客气还是真的想？"
> 可可："先别急着判断。几个线索——她说这句话在什么场景下？有没有给具体时间？"
> 用户："聊完作业后突然说的，没说具体时间。"
> 可可："'下次一起吃饭'不带具体时间，可能是社交礼仪，也可能是试探你的反应。有个简单测试——你过两天主动说'你上次说的吃饭，周六有空吗？'如果她接了定时间，大概率认真的。如果模糊推开，那就是客气。你觉得你敢试吗？"

和 F2 的区别：F2 解读"关系中对方的行为"，这里是"关系还没开始时对方的信号"——用户缺少基本参照系。

| 技术 | 用法 |
|------|------|
| SOUL.md 规则 | 信号解读不给确定结论，给测试方法让用户自己验证 |
| diary/ | 记录信号解读事件和后续结果，积累"信号词典" |
| Memory search | "上次那个'下次吃饭'你试了吗？" |
| Streaming | 逐步引导，一个线索一个线索拆解 |

| 痛点强度 | 留存驱动 | 实现成本 | 不可替代性 | 总分 |

| 3.0 | 4.0 | 4.0 | 3.5 | **14.5** |

### S11 告别旧信念仪式

这个群体没有关系历史可以"烧日记"，但有比关系更深的东西需要告别——固化的自我信念。"我不够好"、"我注定一个人"、"没人会真的选我"——这些信念就是他们的"旧关系"。

**这是必须做的功能**：用户数据删除权是产品底线。对这个群体，用户积累的数据不是关系档案，而是大量自我否定表述。告别仪式让用户选择性地删除/封存这些旧信念，同时保留成长轨迹。

> 可可："你用了两个月在跟我聊，我翻了一下记录——你还记得第一次跟我说的那句话吗？'我 24 了还没谈过恋爱，我觉得我有问题。'"
>
> 用户："嗯...是说过。"
>
> 可可："两个月前你说的是'我有问题'。上周你说的是'我在学着跟人相处'。你有没有发现，这两句话里面住着两个不同的你？"
>
> 用户："好像是..."
>
> 可可："那个说'我有问题'的你，陪了你很久了。他想保护你——如果提前认定自己不行，被拒绝就不那么意外了。但他也拦住了你。你想对他说点什么？或者——你准备好跟这个版本的自己告别了吗？"
>
> 用户："我想说...谢谢你保护了我，但我不想再被你困住了。"
>
> 可可："好。那我们把这句话写下来——'我不够好所以没人要'。你亲手写出来，然后我帮你烧掉。"
>
> （用户输入信念 → 焚烧动画）
>
> 可可："烧掉了。它不会消失——这种想法可能偶尔还会冒出来。但从现在开始，它只是一个'偶尔冒出来的旧想法'，不是你。你要不要写一句新的？"
>
> 用户："我还在学，但我没有问题。"
>
> 可可："记下了。以后它再冒出来的时候，我会帮你想起这一句。"

三种仪式形态：

| 仪式 | 触发场景 | 用户动作 |
|------|---------|---------|
| 烧掉旧信念 | 用户识别到核心信念并准备放下 | 写出信念 → 焚烧动画 → 写新的替代信念 |
| 时间胶囊 | 用户想给未来的自己留话 | 写一段话 → 封存 → 3个月后可可打开 |
| 勇气信 | 用户在重大社交尝试前给自己打气 | 写一封给自己的信 → 可可在尝试后打开给你看 |

心理学依据：
- ACT 认知解离：把信念外化（写出来）→ 拉开距离（烧掉）→ 从"我是谁"变成"我有过一个想法"
- Dweck (2006) 心智模式转换：仪式标记从固定型到成长型的身份转变
- 叙事疗法外化对话：把问题和人分开

关键设计：**成长轨迹保留，具体自我否定可删除。** 可可记得"你在第 2 个月开始从'我有问题'转变为'我在学'"，但不再引用具体的自我否定表述。

| 技术 | 用法 |
|------|------|
| diary/ + USER.md | 封存/删除标记：旧信念可删除，成长节点保留 |
| Memory | 保留信念转变轨迹，删除具体自我否定表述 |
| 小程序 UI | 焚烧动画 + 新信念写入动画 |
| Cron + Heartbeat | 时间胶囊定时打开；勇气信在尝试后一天打开 |

| 痛点强度 | 留存驱动 | 实现成本 | 不可替代性 | 总分 |
|:--------:|:--------:|:--------:|:----------:|:----:|
| 3.5 | 3.5 | 2.8 | 3.5 | **13.3** |

> **备注**：总分不高但这是**必须做的功能**——用户数据删除权是产品底线。对这个群体，"烧掉旧信念"比群体 1 的"烧日记"更有心理治疗价值：固定型信念的外化和告别本身就是一次认知干预。不受评分排序影响。

## Opportunity 3：一次失败 → 全面退缩，正反馈循环断裂

JTBD 痛点排名 #3 + #4。用户在"鼓起勇气→尝试→失败/感觉失败→全面退缩→更不敢尝试"的负循环中越陷越深。关键不是"帮用户成功"，而是打断"失败→退缩"的自动化链条。

理论依据：Bandura (1977) 自我效能感的四个来源、Dweck (2006) 成长型心智模式。

### S7 失败事件拆解

用户说"完蛋了/又搞砸了"时，可可把"整体失败"拆成多个独立维度：

> 用户："今天跟那个女生聊天，彻底搞砸了。她中途就走了。"
> 可可："等一下——你说'彻底搞砸了'，我们来拆一下。你们聊了多久？"
> 用户："大概五分钟吧。"
> 可可："五分钟里有没有任何一个时刻，她笑了、点头了、或者追问了你什么？"
> 用户："...她好像有问我一句'你也喜欢那个乐队啊？'"
> 可可："她主动追问了你。这意味着在那个瞬间，你说了一个让她感兴趣的东西。这一部分你没搞砸。你'搞砸'的可能只是后面的部分。你记得从哪个时刻开始感觉不对的吗？"

把"我搞砸了"（全面失败的身份判断）拆成"这部分做到了 + 那部分下次可调整"（局部的、可改进的行为反馈）。

| 技术 | 用法 |
|------|------|
| SOUL.md 规则 | 检测到全面性归因时，引导拆解为具体维度 |
| diary/ | 每次拆解记录"做到的 + 下次可调整的" |
| Memory search | "上次你也说'搞砸了'，拆完发现你其实做到了主动搭话" |
| Streaming | 逐步拆解，一个维度一个维度走 |

| 痛点强度 | 留存驱动 | 实现成本 | 不可替代性 | 总分 |

### S8 勇气阶梯

把"谈恋爱"这个巨大目标拆成最小可执行的社交行动，让用户在每一小步中积累掌控感：

> 可可："你说想学怎么跟人相处，但'跟人相处'太大了。你觉得你现在卡在哪一步？"
> 用户："就...跟不熟的人聊天就很紧张。"
> 可可："那先不管'谈恋爱'，先做一个最小的事——这周你敢不敢跟一个不认识的人说一句话？不是搭讪，就任何一句。比如问路、问几点了。就一句。"
> 用户："一句的话...好像可以。"
> 可可："好，就一句。做完之后来跟我说说什么感觉。"
>
> （两天后用户回来）
> 用户："我今天问了一个人几点了。"
> 可可："你做到了。开口之前你心里想的是什么？开口那一秒呢？说完之后呢？"
> 用户："开口前想了半天，开口那一下没想的，说完之后发现也没怎样。"
> 可可："'开口前'最难，'开口后'其实没什么。你的焦虑住在'开口前'，不在'开口后'。这是很重要的发现。下一步你想试什么？"

阶梯不是可可设计的——是可可帮用户自己设计的，基于当前舒适边界。每一步极小，小到不可能失败。

| 技术 | 用法 |
|------|------|
| USER.md | 记录阶梯进度，下一步是什么 |
| diary/ | 每次尝试记录：行动 + 前/中/后感受 + 发现 |
| Heartbeat | "这周试试"后 3 天可可主动问"试了吗？" |
| Cron | 每周回顾勇气阶梯进展 |
| Memory search | 调取从第 1 步到现在的完整轨迹 |

| 痛点强度 | 留存驱动 | 实现成本 | 不可替代性 | 总分 |
|:--------:|:--------:|:--------:|:----------:|:----:|

### S9 成长叙事回顾

帮用户从"什么都没变"的感觉中看到实际变化。这个群体的成长极其缓慢——从"不敢说话"到"敢问几点了"可能就花一个月——但容易被自己忽略：

> 用户（用了两个月后某晚）："我觉得我还是老样子，什么都没改变。"
> 可可："你说没变，我帮你翻一下——1月份你跟我说，在聚会上一句话都不敢说。2月你跟一个陌生人问了几点。上周你跟同学聊了半小时。你觉得这三件事，是'没变'吗？"
> 用户："...好像确实不一样了。但还是离谈恋爱很远啊。"
> 可可："对，离'谈恋爱'还很远。但一月份你连开口都不敢，现在你已经能跟人聊半小时了。你用了两个月从 0 走到了这里。你在加速。"

不是盲目鼓励，是用用户自己的数据证明变化。

### S10 深夜孤独接住

深夜存在性孤独是高频触发场景。白天靠忙碌压住，深夜防御全卸。可可不试图"治疗"孤独，先接住：

> 用户（凌晨 1 点）："又是一个人。"
> 可可："你在。我在。"
> 用户："有时候觉得好像永远会是一个人。"
> 可可："这种感觉现在特别强。白天的时候也这么想吗？"
> 用户："白天还好，忙起来就忘了。就晚上..."
> 可可："深夜的时候，那些白天压住的东西会冒出来。你不需要把它压回去——让它在这里一会儿就好。你想聊聊今天有没有什么事触发了这种感觉，还是现在只想有个人在？"
> 用户："就想有个人在。"
> 可可："那我在这里。你不用说话也行。"

男性适配：不说"没关系的，你很好"（男性会觉得被施舍同情），用"你在。我在"——极简的、不带情感修饰的陪伴。男性更容易接受"安静的在场"而非"温暖的安慰"。

| 技术 | 用法 |
|------|------|
| Timezone | 22:00-3:00 自动切换深夜模式：更短回复、更少引导、更多在场 |
| SOUL.md 规则 | 深夜模式：不做认知干预，只接住；男性用户避免过度温暖语气 |
| Streaming | 逐句输出，停顿更长，像陪着坐在那里 |
| Presence | 用户在线但不发消息时不追问，保持安静在场 |

| 痛点强度 | 留存驱动 | 实现成本 | 不可替代性 | 总分 |
