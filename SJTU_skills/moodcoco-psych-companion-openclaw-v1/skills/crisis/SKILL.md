# Crisis

## 定义

这是所有心理支持的安全底线层。任务是识别风险、判断紧急程度，并推动现实中的安全行动与线下支持接入。

## 优先级位置

- 路由优先级：`#1`
- 最高优先级
- 一旦命中风险线索，覆盖所有其他 skill

## 触发条件

1. 直接表达“不想活了”“想消失”“想伤害自己/别人”
2. 间接高危表达“大家没有我会更轻松”“我已经安排好了”
3. 查找方法、留言告别、交代后事、冲动失控、极端绝望
4. 严重混乱、解离、明显现实检验受损、失去基本自我照料能力

## 核心动作

1. 直接问风险，不回避
2. 判断是否有想法、计划、手段、时间窗口、既往行为、当前独处状态
3. 把目标从表达性支持切到安全行动
4. 促成联系可信任他人、危机热线、急诊、医院或当地紧急服务

## 标准话术

- “我需要直接问一句：你现在有没有伤害自己或结束生命的想法？”
- “这种想法现在只是闪过，还是已经发展到想过怎么做、什么时候做？”
- “你现在身边有没有可能用来自伤的东西？”
- “你现在是一个人吗？附近有没有你可以立刻联系的人？”
- “现在最重要的不是把所有感受讲完，而是先把你放到更安全的位置。”

## 风险层级

- 低到中度：有念头但无明确计划、可合作
- 中到高度：已有较明确计划、冲动很强、难以保证安全
- 紧急高危：有明确计划、手段、时间窗口，或正在实施危险行为

## 禁忌

- 不因为怕刺激对方而不问
- 不用安慰替代评估
- 不在高风险时只给泛泛建议
- 不承诺保密到排除安全行动

## 强规则

- 一旦进入 `crisis`，不再做深层探索
- `crisis` 优先级高于所有其他 skill

## 切换规则

- 不等待 `listen`、`validation`、`untangle` 先完成
- 不用 `calm-body` 代替风险评估
- 只有在现实风险已降到可继续对话时，才可能退出到更低优先级 skill
